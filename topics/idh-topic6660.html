Service History

Subject: Issue on NS7080W S/N:121080 at Xilinx

Handler Model: NS-7080W (Xilinx)

Controller: RC520

Date: 29 Apr 2008


 

Symptom

a) NS7KW-01 – Upgraded to new software G8.10-SXC , having numerous Automat error, causing overspeed in low power state error. (Recover by POWER OFF/ON).

b) NS7KW-02, 03 & 04 – pause start, hang up running G6.10-SXC. CPU execution issue causing hangup? Should we upgrade RAM? XAP handler having numerous interfacing, eg. in line scan etc (for NS7KW-01,02,03 &04). Only 128MB RAM currently on motherboard. Should we upgrade to 512MB?


 

Action

NS7KW-01-Handler hang up and automat error

Motor tunning done since 29/04/08. No more automat error. 13/05/08, fine tune again for optimised performance and will monitor.

Software patch since 05/05/08, no more hang up of G-7101 "UNKNOWN ERROR" based on error log.  This G-7101 "UNKNOWN ERROR" is actually the "Quick Pause DAEMON 07 Failed (Error:7101)", The error came out due to PC was busy.
When you operate too fast, PC will not be able to proceed it then this error come out. It is like hang/jam in PC. The patch file from EPSON is Xilinx.reg

The procedure to apply Xilinx.reg patch is as below:

 1. Closed MMI.
 2. To copy the attached file to FD or Thumb drive and insert to Handler.
 3. Double click the file. Pop up window(Registry Editor) appear, and press "OK".
 4. Start  MMI.

 This file will be reduce the error.
NS7KW-03-False device drop alarm

Verified ARM 1 vaccum sensitivity w.r.t "1 1/2 turn" concept.

Adjust counter air purge due to output shuttle device flip.

Perform Index Arm height calibration on shuttle/socket. Running check with dummy units – O.K


 

NS7KW-06-Device scratch issue, suspected index alignment out

As it happened on AP kit, we're unable to look further so used KA kit to verify handler alignment.

P21, P22, P23, P24, P50, P51 verification. Less than 0.2mm. Condition "ok".

Just a note:

Change kit used for NS7KW-03 was from AP. Significant play of the device in the contact head was noticed after device pick up from shuttle. This play MAY cause device scratch issue. For the device scratch issue on NS7KW-06, there are many factors to look at. Change kit from AP, we're unable to advise/comment further.

Remarks

There was a case of device drop for both Index Arm 1 & 2, after replacement of Remote I/O Board, problem solved.


 

Check points for Index Arm Device drop:

- vacuum cup condition, device drop more frequently in hot mode?

- o-ring worn out on plunger

-Check diameter/tolerance of guide pins on socket guide plate

-Check any broken/bent tubing or tubing pressed by cable tie on SLK

-Check air fitting/elbow or gasket not loose on SLK

-Check if guide pins bent/worn out

-Check vacuum filter dirty

-Check vacuum filter put properly

-Check seal for vacuum put properly and not torn

-Check plastic casing for vacuum filter not broken

-Check nest guide pin's hole

-Check o-ring on SLK

-Check metal hooks of Socket Layout Kit (SLK)

-Check correct brand and specification of latch use for (Index) Test Hands

-Check metal latch worn out at Index Test Hand

-Check Index teaching position P50 and P51

-Check if Index Arm crash before or Index Arm bent(Index Arm alignment check)

- if device never drop, adjust vacuum switch sensitivity


 

Correct metal latch of parts No is TL-29A, this is supplied by Tochigiya.
