Service History

Subject: UBS During recovery encounter blue screen with *inaccessible boot device* error

Handler Model: UBS (UBS-02)

Controller: RC520

Date: 26 Feb 2010


 

Symptom

UBS-02 HDD crash.

Unable to use UBS-12 EPSON Recovery CD as blue screen with *inaccessible boot device* error message.

Blue screen with *inaccessible boot device* error is a symptom of using wrong Recovery CD due to wrong CPU Board.

USB-02 could be using PC-686BX which does not have EPSON Recovery CD.

Only manual recovery by install Windows 2000 and etc.


 


 


 

Action

Used UBS-15 CU with UBS-02 HDD, recovery OK and able to boot up.

MMI software version up and SPEL restore project and system file.

Uni-wire disconnection error message.

Already selected tray drawer. Suspect physical wires/connection wire not present and advise to reset Remote I/O board(A128-01-Y03) and restart. OK.

LED display does not display binning. Suspect CONTEC PO-64L(PCI) PO card address different between SPEL and windows since recovery is for UBS-12.

CONTEC PO-64L(PCI) address in SPEL Harware I/O Board must be the same as in Windows 2000 address

(Device Manager->Multifunction Adapter -> Contec Co.,Ltd-PO64L(PCI); Properties-Resources-Input/Output Range)


 

Found SPEL address different when compare with other handler. Set address according to UBS-3 and UBS-33 and ok.


 


 

Cause


 


 

Remarks

UBS with CPU Board SPI-6941 will have EPSON Recovery CD.

UBS with CPU Board PC-686BX will not have EPSON Recovery CD, only method is by manually install software such as Windows 2000 and etc.

Maybe USB#12 and below are using old type CPU Board PC-686BX.
