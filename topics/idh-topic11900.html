Subject: EEMS- NS8080 Device not placed on hotplate in HOT Mode.

Handler Model: NS-8080 (S/N: 181655)

Controller: RC520

Date: 28 Feb 2010


 

Symptom

When running 1x2 and 0ne site was disabled, only first two devices were placed on the hotplate. Subsequently, the input arm just pickup device from loader and it didn't release to hotplate instead it release on  shuttle.

Tried creating new setup file from Epson default setup file but still the same.

  -Running 1x2, 2x2, busy shuttle without disable, after clean out 1 device left on hot

plate.

-Disable 1 site , after clean out the index arm 2 hang during contact with socket without error message.

Cause


 


 

Remarks

1 Mar 2010 email

EPSON has found some bugs in HMI(Ver 1.0 or 1.1-sp) for NS-8080 8-hand which will cause problems to the setup running 1x2(disable 1 site).

The patch will be release by this week according to Hidaka san.


 

4 Mar 2010

UTAC report same problem as EEMS today when the handler was running hot temp, 1x2 disabled 1 site. Hidaka san and I confirmed on the handler and upgraded the HMI from 1.0-sp to ver 1.3.

 

We also upgrade NS8080-01 from ver 1.1 to ver 1.3 when checking on the input vacuum sensor off error issue.

 

However, NS8080-03 unable to upgrade due to the handler running hot device and no system time. HMI is still ver 1.1.
