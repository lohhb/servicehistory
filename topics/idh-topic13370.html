Subject: UTAC NS8040/NS8080 hotplate odd number 3x7 matrix issue

Handler Model: NS-8000 series

Controller: RC520

Date: 1 Jul 2010


 

Symptom

Using odd number Hotplate matrix for NS-8040/NS-8080.


 


 

Action

Try out on NS8080-12.

When check(tick) "Use All Pockets"  option in NStask bar [Maintenance]->[maintenance]->[Configuration] of HMI, then can key in odd number matrix for hotplates in setup file.

But only 37.5x37.5 can run.

Need to use Z-mode for input pick up.

Random mode will encounter RC520 error at hotplate.


 

For 40x40 and 45x45, both Z-mode and random mode will encounter RC520 error at loader and hotplate respectively. Already feedback to Hidaka-san, awaiting Japan reply.
