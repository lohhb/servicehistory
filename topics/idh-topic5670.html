Service History

Subject: Device drop at Index Arm 2-A

Handler Model: NS-7080W (Xilinx, S/N: 121266)

Controller: RC520

Date: 25 Jan 2008


 

Symptom

Set-up the handler for running TQFP144 20x20 package. Auto Contact height carried out on these new package and subsequently followed by a fine tune for these package. During the dummy run, there were a number of device drop errors at Index Arm 2-A.


 

Action

Investigation and the checks are as follow:-

a) Index Arm 1 & 2 vacuum sensitivity check carried out found good, but vacuum Index Arm 2-A is not as strong as other hands, further investigation found that the nuts for Index Arm 2-A tubing is not tightened, after tightening, problem still persist, other than Arm 2-A, a side finding is that one nut is not even tightened, only a few threads tightened, due to time constraint, only tightened those nuts that are accessible,

b) nest for Index Arm 2-A swapped with Index Arm 2-C, did a device run, problem still persist,

c) changed to a new set of layout kit, problem still persist,

d) swapped input shutter 1 and 2, problem still persist,

e) vacuum wait time increased (0.8 to 1.0), problem still persist,

f) pick up height lowered for index arm 2 (33.5 slowly reduce to 32.9), problem still persist,

g) when problem occurred, take out the affected device and swap another device into pocket A on input shutter 1, "RETRY", "START", sometimes will pick up the swapped device, but most of the times problem still exist.

h) pick up height raised for index arm 2 (33.5 to 34), problem solved. After solving index arm 2, the same problem happened on index arm 1 hand D, so pick up height raised for index arm 1 (33.7 to 34.1) and problem also solved, and

i) run 2 trays of dummy, above problem did not appear, thus handler released to them for ICOS verification which will be carried out tomorrow, 26/01/08.


 

Cause

Pickup height from input shuttle for Index Arm 1 & 2 is too low.


 


 

Remarks

There was a case of device drop for both Index Arm 1 & 2, after replacement of Remote I/O Board, problem solved.


 

Check points for Index Arm Device drop:

- vacuum cup condition, device drop more frequently in hot mode?

- o-ring worn out on plunger

-Check diameter/tolerance of guide pins on socket guide plate

-Check any broken/bent tubing or tubing pressed by cable tie on SLK

-Check air fitting/elbow or gasket not loose on SLK

-Check if guide pins bent/worn out

-Check vacuum filter dirty

-Check vacuum filter put properly

-Check seal for vacuum put properly and not torn

-Check plastic casing for vacuum filter not broken

-Check nest guide pin's hole

-Check o-ring on SLK

-Check metal hooks of Socket Layout Kit (SLK)

-Check correct brand and specification of latch use for (Index) Test Hands

-Check metal latch worn out at Index Test Hand

-Check Index teaching position P50 and P51

-Check if Index Arm crash before or Index Arm bent(Index Arm alignment check)

- if device never drop, adjust vacuum switch sensitivity


 

Correct metal latch of parts No is TL-29A, this is supplied by Tochigiya.
