Service History

Subject: Unable to start HMI

Handler Model: NS-8000 series

Controller: RC520

Date:


 

Symptom

Unable to start HMI.


 

Case 1 )

System check error.

Can't detect for HDD.


 

Case 2)

The monitor screen is dark (no display) after starting Windows.


 

Case 3)

The monitor screen has no display.


 

Case 4)

When HMI starts, only show NSTask bar.

Display [SPEL executing].


 

Case 5)

During HMI loading, Database open error(connect) - ODBC error.

Open Observer Data, encounter Runtime error for Microsoft jet data data base engine


 


 


 

Action

Case 1)

Replace Hard disk (HDD).

Refer Maintenance Manual Chapter 8.


 

Case 2)

Replace Hard disk (HDD).

Refer Maintenance Manual Chapter 8.


 

Case 3)

Replace CPU Board.

Refer Maintenance Manual Chapter 7.3.


 

Case 4)

End task(by Ctr-Alt-Del).

Start SPEL manually.

Start HMI manually.

Delete current setup file and select another setup file.


 

Case 5)

Close HMI.

Copy & paste handler.dsn and handler.mdb in C:\NS800\Env from another handler.

Launch HMI


 


 

Cause

Case 1) HDD spoilt.


 

Case 2) HDD spoilt.


 

Case 3) CPU Board spoilt.


 

Case 4) Setup file corrupted and caused hang up.


 

Case 5) Handler database file handler.mdb corrupted in C:\NS800\Env
