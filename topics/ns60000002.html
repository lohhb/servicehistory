Service History

Subject: Output Arm Hand up error

Handler Model: NS-6040

Controller: SRC-520CE

Date: 31 Dec 2007


 

Symptom

After press HOME for first time NS-6040 able to HOME.

The second time press HOME, after Input Arm HOME completed, Output Arm's Hand 1 & 2 move up and down one time then up and down again, MMI issue HOME sensor did not turn off, Output Arm X & Y Axis did not move.

In SPEL 95 ver 2.5, able to MCAL or HOME for robot 3(output Arm), this was tried out many times.


 


 

Action

- Check whether output arm Hand 1 & 2 Hand Up sensor(HOME) LED cut off when Hand 1 & 2 are pull up to cut it's sensor, ok


 

- In SPEL I/O monitor, Output Hand 1 & 2's Hand up sensor (HOME) display on screen did not turn on when the Output Hand 1 & 2 are at it's HOME position.

Output Hand 1 Hand up sensor is In 58, Output Hand 2 Hand up sensor is In 59.


 

Similarly, in I/O monitor, for Input Arm, it Hand 1 & 2's Hand up sensor display on screen did not turn on when Input Hand 1& 2 are at it's HOME position.

Input Hand 1 Hand up sensor is In 68, Input Hand 2 Hand up sensor is In 70.


 

- Check I/O connector I/O 1 and I/O 2 connectors that are behind the SRC520CE controller, found that I/O connector I/O 1 and I/O 2 connectors were connected to the wrong port.


 


 

Cause

Found out that the I/O connector I/O 1 and I/O 2 connectors were connected wrongly behind the SRC520CE controller.


 


 

Remarks

I/O 1 connect to

- Index I/O (IN 48 ~ 55, OUT 48 ~ 63)

- Input Arm I/O (IN 64 ~ 79, OUT 64 ~ 78)

- Output Arm I/O (IN 56~63)


 

Same situation as 0013-NS6K.

I/O 2 connect to

-Output Arm Out (IN 58 ~ 63, OUT 80 ~ 95)

- Shuttle In (IN 80 ~ 111)
