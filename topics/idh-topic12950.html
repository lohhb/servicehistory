Service History

Subject: STM NS-8080 Octal site double unit sensor setup unable to communicate when using Advan type 1 for GPIB

Handler Model: NS-8080 (S/N: 181581)

Controller: RC520

Date: 9 Jul 2010


 

Symptom

NS-8080 octal site conversion.
 -Currently handler GP-IB tester interface  is using "Advan type 1" for
 Quad site; found that it is not workable for Octal site.
 After changing to Epson type M and Epson Type K, the "ERROR" Bin was
 solved, and successfully got site 1 to 8 binning.
 Please refer to the attached file for the data collected  from I/F monitor.
 -GP-IB tester interface Epson type M and Epson type K is only passed
 all the binning for "testing" site mapping as below
 1 2 3 4
 5 6 7 8

 

-But for the actual load board site mapping that is still failed with
 "ERROR" Bin as below
 5 1 3 8
 6 2 4 7

 

-For the GP-IB tester interface which is Epson type M and Epson type K
 is able to run on octal site,


 


 

Action

After some trial runs, the tester is able to communicate with handler using 'ADVAN type 1' GPIB settings with load board 8 sites mapping as
 below,
 5 1 3 8
 6 2 4 7
 -According to Carol, he just reinstalled the tester UI.


 


 

Cause

Tester program was not compiled for Octal site(after conversion from Quad site) causing tester unable to communicate with NS-8080 when using  'ADVAN type 1' GPIB setting.

Tester require reboot to compile for octal site running.


 

Remarks

-For the GP-IB tester interface which is Epson type M and Epson type K is able to run on octal site, how do I know which is the recommended setting for running octal sites?


 

-I did check with EEMS, they were able to run well on octal site which their GP-IB interface is using Advan type 1.

But how come STM failed on it?
