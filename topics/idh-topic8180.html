Service History

Subject:

- PSUNT 24-2V Fuse tripped (CB13)

-T-4040 Communication error with the pulse generating board

-T-4301 Pulse Generating Board detected a limit signal

- MMI display "Power down"

Handler Model: NS-7000 (S/N:18732)

Controller: Rc520

Date: 10 Feb 2009


 

Symptom

Silabs' NS-7000 S/N:18732 is having some intermitten problem.

During operation, the 24-2V CB13 fuse at PSUNT will trip and error message appear in MMI as " T4301 Pulse Generating Board detected a limit signal" and MMI status display as "Power down". We had checked all Input/ Output Arms' stepping motors(actuator motors and open/close motors) and couldn't find any presence of short circuit.

 

Handler is still able to resume operation by turning off power from panel and reset 24-2V fuse at PSUNT. However, error still occur after running for about an 1hr.


 

Encountered T 4040 Communication error with the pulse generating board and T 4301 the pulse generating board detected a limit signal.

24vdc CB13 will trip when error occurs but able to reset after OFF/ON operation panel.


 


 


 

Action

Checked connectors behind RC520 controller a bit loose and tighten.

Checked input and output hand motor and sensor cables. No abnormalities.

Error occur intermittently.


 


 

Cause


 


 

Remarks

24V-2 supply voltage to stepper driver for Input/Output and Open/Close Hands.

Refer Electrical circuit main block diagram in Wiring manual.


 

PG issues follow up with 0150-NS7K, 0151-NS7K and 0161-NS7K.


 

PG Board C-863 and C-864v1 are not compatible.

PG#1 control Z motion of Input Hand A, B, C and D.

PG#2 control Z motion of Output Hand A, B, C and D.

PG#3 control Open/Close motion of Input Hand and Output.
