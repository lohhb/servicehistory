Service History

Subject: Tester Time up error

Handler Model: NX1032XS (S/N: 182347)

Controller: RC520 (S/N: 00504), DUBOX030 (S/N: 100004), PSUNT353 (P892)

Date: 1 Nov 2012


 

Symptom

Online Amb run Binning check FF900 31x31 Octal but disable site B & H1st round, 1st & 2nd device test bin to bin 1 but 3rd device testing tester time up error.Clear out devices2nd round, 1st & 2nd device test bin to bin 1 but 3rd device testing tester time up error.


 

SPEL software corruption after IFTUNT300 driver upgrade from ver 1030 to 1040

After reboot, “Retrun value is 0” error message was displayed during launching of SPEL.Also SPEL hangup.


 


 

Action

HDD Recovery and system data, pos and setupfiles restore due to SPEL software corruption after IFUNT driver upgrade from ver 1030 to 1040.

We performed the following:

- Use Recovery DVD Ver 1.0 to do HDD Recovery

- HMI upgrade to ver 1.41

- HMI System Restore

- HMI Pos Restore

- HMI setup files restore by importing


 

- Running check with devices but found device pickup error at loader and also at output shuttles. After found X & Y pickup & release offset was out

- After advise by Shimizu-san to copy PG parameters from backup to C:\NX1000\Config\PG. Backup of PG parameters were copies to C:\NX100\Config\PG

When doing HMI SYSTEM or SPEL SYSTEM backup, the PG Parameters are not included.

But yet PG setting such as it's address are in SPEL.

- Restore backup setup files and running check with devices, pick & place offset, ok


 

IFUNT driver downgrade to ver 1030.


 

4 Nov 2012

EPSON release HMI ver 1.42 to patch Tester communication issue(test end at the tester communication).


 


 

Cause

HMI ver 1.41 bug caused Tester communication issue.

HMI ver 1.42 (SPEL-CT Ver3.20-h2) patch this bug.
