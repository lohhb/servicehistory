Subject: NS-8040 After power up encountered Uniwire disconnection error

Handler Model: NS-8040 (S/N: 181394)

Controller: RC520

Date: 12 Mar 2010


 

Symptom

Power up handler, encountered "Disconnection of uni-wire1 transmission line or terminal problem".


 


 

Action

Remedy by HMI software states (1) Remove the cause of the problem of uni-wire No1.

I performed the following:

- reset Remote I/O board, pressed HOME but HMI status windows indicate please close Index side door

- found at rear top-left of Index, 2 I/O modules no power, no light.

- open top cover of Index and resit all I/O modules black power connectors

- reset Remote I/O board Able to HOME handler


 


 

Cause

Loose connection on 2 I/O Modules behind hander rear-left Index.


 


 

Remarks

Just check all of uniwire wiring condition.

 (Check uniwire LED is turned on or off one by one.)


 

 There is no identify which uniwire is problem because of mechanical limitation.


 

 Note: Uniwire-01 =  No.1 Remote I/O board happen error


 

          Uniwire-02 = No.2 Remote I/O board happen error


 

          In case of NS-8160W and 8160SW, 2 sets of Remote I/O are assembled to PC.


 

1) EIN 30 of 16CH (located below Loader unit) include Cover Open Switch(Input Bit 299)

2) EIN7 consist of Input Bit( 104~110 connected to SKP452 E-stop board) for Doors

3) EIN30 of 8 CH (located above Index) include Input Bit 284 for Index side cover door
