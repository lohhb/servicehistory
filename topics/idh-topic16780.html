Service History

Subject: NS8080 High UPH 8 Hands check and investigation by EPSON member

Handler Model: NS-8080 (S/N: 181211) High UPH 8 Hands

Controller: RC520 (S/N: 03-00507)

Date: 21 Jun2011


 


 

Symptom

 NS8080 High UPH 8 Hands check and investigation by EPSON member.


 


 

Action

1. NS8080-01 8 I/O check and investigation by EPSON member.

- Check z-axis movement of all hands by panel power off.

- Check input and output hand height by Z Up Down in autoteaching.exe.

- Check zteach home sensor on/off pulses.

- Collected config, log and observer data.


 

Findings:

a. Input arm – Mechanically ok. Abnormal HOFS value.


 

 Action – Performed master calibration.


 

b. Output arm – Persistent Hand H hand up sensor error. Hand C,D and F vibration issue. Hand G ball spline shaft NG.


 

Action – Resit Hand H sensor connector and replaced connector brown, white, blue molex pin but problem still persisted. Tidy up home sensor cable, able to resume operation temporary.

 – Ikeyama san arranged replacement z-axis module for Hand C,D,F,G,H. ETA this Fri 24 Jun.


 

2. NS8080-03 8 I/O check and investigation by EPSON member.

- Check z-axis movement of all hands by panel power off.

- Check input and output hand height by Z Up Down in autoteaching.exe.

- Check zteach home sensor on/off pulses.

- Collected config, log and observer data.


 

Findings:

a. Input arm/output arm – Mechanically ok. HOFS and calibration data ok.


 

Action – To discuss with mechanical enginner regarding spline shaft o-ring and home sensor connector connection issue.


 

Additional information:
1. Vibration issue due to assembly problem. Possible to remove hand and repair. Need further details.

2. Two possible causes to Hand up sensor error issue. Connection problem of connector and hand height difference. Need to perform master calibration whenever hand parts replaced. Need to perform hand height calibration whenever replacement of changeover kit.

3. Two types of z-axis module. Red color motor on NS8080-03 and NS8080-05, grey color motor on NS8080-01 and NS8080-02. Red color = new type, grey color = old type. Difference on the ball spline shaft with improved smoothness. Ball spline shaft o-ring material also improved on new type.


 


 


 

Cause


 


 

Remarks

a. Explanation of RS485 2Ack, command, unknown, servo and hand up sensor errors by Ikeyama-san and Miyamoto-san.

- RS485 2Ack : PC connected to I/O arm by RS485 communication lines. PC send command to I/O arm, I/O arm send back status to PC. Then I/O arm moves to target position and send status again to PC. If 2nd status not received by PC, 2Ack error occurs.

- RS485 command error : PC send command to I/O arm and wait for status reply. If any system malfunction occurs, and unknown command sent to I/O arm, I/O arm unable to read command and command error occurs.

- RS485 unknown error : Total 20 errors of motor. 16 of them are important errors so they will show up as unknown errors.

- Servo/hand up sensor error : PC connected to I/O arm by 2 I/O lines. 1 is servo, 1 is home sensor. If PC unable to received status from either line, error occurs.


 

b. Presentation of checking results of 8 I/O handlers (#1, 2, 3, 5) from 21-22 Jun, by Imai-san.

- For NS8080-01, output arm hand c,d,f,g found to be not smooth. Replacement part ETA 24 Jun. UTAC arrange system time next week.

- For NS8080-02, input arm hand c,d,e,f and output arm hand c,d,g,h found to be a little not smooth. Temporary adjusted.

- All remaining grey color type motor z-axis module to be replace by new red color type. ETA of parts to be advised.

- Getech provide past 8 I/O warranty replacement record and remaining hands not replaced for NS8080-01 and 02.


 

c. Analysis of error of 8 I/O hands by Ikeyama-san.

- NS8080-01 output hand H home sensor cable found to be longer. Suspected internal wire broken. To inform root cause end of next week(July 1).

- 2 causes of hand up sensor error. Master calibration not done after hand replacement and sensor cable internal broken.

- Cause of z-axis servo error. Tolerance of fixture between linear motor and ball spline is big. If abnormal force applied to hand, motor shaft and ball shaft would not be parallel.

- Tolerance of fixture already tightened. To replace z-axis modules which uses old fixture.

- Servo error recovery issue. All 4 handlers already upgraded to HMI v1.9. Recover error by pause, reset, cont start instead of pressing OFF button on operation panel.

- If master calibration not done after motor replacement, the pick up position is changed and abnormal force will be added on the z-axis motor.

 

d. ATC issues.

- Lots of problem on Silicon thermal ATC.

- Ikeyama-san to provide information of EPSON integrated ATC system.
