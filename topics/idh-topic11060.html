Service History

Subject: Qualcomm NS-8080HW hang up during HOME

Handler Model: NS-8080HW

Controller: RC520

Date: 15~16 Oct 2009


 

Symptom

Handler hang up during HOME.


 


 

Action

1.   In SPEL CT, Individual mcal of each robot.

2.   Found Input arm unable to mcal.

3.   Found Hand A has no torque after power on.

4.   Swap motor connector at SKP-456G between Hand A and Hand B. Problem migrated to Hand B. Conclusion Motor A is ok.

5.   Swap connectors at Drv12 and Drv13. Problem did not migrate. Conclusion Drv 12 is ok except the path after SKP-430 J1 to Drv12 and Drv12 to SKP-456G.

6.   To Verify 12CN3 and 13CN3 connector swap. If migrate, possibly SKP-430 J1 problem. If still Hand A, path after Drv12 to SKP-456G including SKP-456F.

7.   Swapped 12NC3 and 13cN3, problem did not migrate. Confirmed path is after Drv12 to SKP456G.

8.   Replaced new SKP456G. Problem did not migrate.

9.   Replaced new Stepper motor. Problem did not migrate.

10. However, today different symptoms. Hand C and D going down and Hand A and B moving up but vibrating.

11. Swapped Drv12 and Drv13. Problem same.

12. Swapped PG board. Problem same.

13. Verified SKP430 connection. No Problem.

14. It seems Input hand all 4 pick up hand has abnormalities. Suspect coming from general source. Attempted auto height calibration but unable to.

15. Checked SKP456F and X1 connector ok, but found X2 connector loosely connected. Resit ensuring both connect is “clipped”, mcal and now all 4 pickup hands no problem.

16. Conclusion, suspect X2 connector, which is the main connector for input arm loosely connected causing abnormalities. After resit with “clipped” dry run no problem.


 


 


 

Cause

X2 connector which is main connector for Input Arm connection loose caused abnormalities include handler hang during Homing.


 

Remarks

Dip switch on stepper driver if wrong setting will cause stepper motor to vibrate and unable to complete HOME or unable to complete Auto Hand Height calibration.
