Subject: NS-8080 4 Input Hand Socket Auto Cleaning Option Issue with Octal disable sites, encountered similar with H02-NS816 PNP program sequence wrong when using auto socket cleaning option

Handler Model: NS-8080

Controller: RC520 HMI ver. P11.51-SAS

Date: 3 Jan 2013


 

Symptom

 NS-8080 4IO Socket Auto Cleaning Option Issue with Octal disable sites encountered similar with H02-NS816 PNP program sequence wrong when using auto socket cleaning option.

Upon verification , we are noticed this error only encounter on hot mode octal site setup when disable site.


 

When clear any error during on the handler has turn on socket auto cleaning option .

>There will be encounter False alarm triggering on hotplate pockets filled with units (Device Remaining Error)

>Remove the devices from hotplates  will be encounter Device Pickup Error On Hotplate.

>Press SKIP button, input arm will place the units on disable site shuttle pocket.

This issue only happen on octal site setup but disable individual site.


 

As feedback by ASE, they are facing issue with auto socket cleaning issue when they are running octal setup with disable sites.

1) No issues when running octal full sites.

2) During socket auto cleaning execution with octal setup with sites disabled, no issue.

3) Feedback by customer after socket auto cleaning, whenever there is any error occur at the loading side will cause error on input arm loading sequence.

4) Issue only happen when running in hot mode.
