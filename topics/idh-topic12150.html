Service History

Subject: AMD – UBS VBLD & 2D reader support for MMI A12.21SAC software modification by Miyasaka-san

Handler Model: UBS

Controller: RC520

Date: 23 & 24 Jan 2013


 

Symptom

AMD – UBS VBLD & 2D reader support for MMI A12.21SAC software modification by Miyasaka-san

MMI A21.21SAC issues:
-Ball Damaged(BD) Total Count alarm does not activate when count reach 2
-After BD error, Pause+Reset followed by Continuous start will bin to wrong binning
-2D reader “Decode Unit : Time Out” error after input arm pickup 20~30 devices


 


 

Action

On UBS15 modified Variable.inc, Const.prg & Outarm.prg for total BD count alarm.
BD counter will only reset to zero when Initial Start or after BD alarm.
Correction to wrong binning.
Interim MMI A12.21A SAC created.
Verification run with package by customer.
On UBS09 simulation for input 2D reader “Decode Unit: Time Out” error using RS232 analyser. (Previously, AMD solved by adding 50ms delay after 2D PC send length (15 char) of bar code to UBS and followed by the 2D barcode.)
Using RS232 Analyser, 2D PC send 15 characters to UBS but UBS received 14 characters.
Sleep 0.5 command removed from OCR.prg and add auto retry to scan 2D and read back to UBS again.
MMI A12.22 SAC created.
Customer will do further verification on this new MMI.
SR: 012518


 


 

Cause


 


 

Remarks

Issue:

AOI (previously call VBLD) BD Counter in Total mode not working in total, but in two sockets consecutive failure to trigger alarm.

When AOI alarm happened, the AOI failed device on output arm binned to final test binning tray instead of fix tray (for AOI failure device only)

The other device on another output arm binned to I/F tray instead of respective tray based on testing result.

UBS sometimes missing 2d result from 2d pc to cause no data received time out error (occurs every 20 to 30 units). This issue has been

settled before Epson engineer come with 50ms delay between sending barcode data length and barcode data result. But Epson still need

investigate why missing to receive 2d result sometimes

Result:

BD Counter issue has been fixed and verified working fine as the way we designed. When AOI alarm happens, only UBS changed to initial start can reset the BD Counter.

When BD Counter reaches and AOI alarmed, output arm AOI failed device binned to fix tray, and the other device on another output arm binned to correct tray based on

it’s final testing result

2D no data received error has been observed by using Epson RS232 analyzer. RS232 analyzer shows 15 characters sent from 2d software (15 is the correct data length of 2d barcode)

But in UBS software, it shows received 14 characters, in order not to interfere operator to click retry start, UBS software make changes to send auto-retry to 2D pc for 2D decoding command

(maximum three times). With both 2d software and UBS software changes regarding this 2D issue, it is more reliable to avoid 2d no data received time out error in production run.

Epson will provide latest ver 13.22 official release to AMD by end of next week (Feb/03/13)

Ver 13.22 is the latest modified UBS software and installed in UBS15 and UBS09, we still need monitor the new software for a few weeks to make sure any new changes will not introduce new

Issue or affect normal testing.
