Service History

Subject: NS-6040 and NS-7000 will retest same device resulting in same ID double testing for Broadcom device.

Handler Model: NS-6000 and NS-7000

Controller:

Date: 4 Feb 2009


 

Symptom

NS-6040 and NS-7000 will retest same device resulting in same ID double testing for Broadcom device.


 

1)Broadcom security devices found same ID double testing from summary using NS6040.


 

2)Error log showed jam occurred during the double testing.

3)Investigation found NS6040 will retest device in index arm during handler jam/pause status.

4)In the event Binning has been received or Binning has not been received, index arm will still test device for NS6040.

5)NS7000 however, if jam/pause status, if binning has been received, will bin according. But if binning has not been received, will retest device.

6)Problem occurs when binning has not been received but tester has finished testing, both NS6040 and NS7000 will retest same device resulting in same ID double testing for Broadcom device.

ASE request software change for the sequencing. Please refer to meeting minutes.


 


 

Action

Device drop in index arm when Home depressed for NS6000(this is normal), but NS7000 is different - To change all NS7000 to behave like NS6000


 

NS-6040 and NS-7000 Before or after receiving “EOT”, all devices must bin as error.

-To change software sequence.


 

NS-7000 Error message after site map change

 - Incorrect display of error msg after user disable/enable site after “pause + reset” during production.
