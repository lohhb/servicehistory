Service History

Subject: NS-7000 Sensor Detection error at Output Shuttle 1

Handler Model: NS-7000 (UNISEM IPOH)

Controller: RC520

Date: 21~25 Feb 2011


 

Symptom

NS-7000 Sensor Detection error at Output Shuttle 1.


 

Action

21 Feb 2011

Replace Shuttle board, check sensor wiring, check sensor alignment.

With side device sensor option checked, both shuttle 1&2 unable to run.

Only possible to run without side sensor for shuttle 2.

Temporary disable shuttle 1.


 

24 & 25 Feb 2011

Side sensor not used, J0505 Detection sensor error on shuttle 1.

Side sensor used, J502 and J503 on both shuttles.

Based on side sensor not used, check timing sensor and top sensor alignment. OK

Checked 16CH Shuttle 2 Input I/O module for Gate Motion Sensor, OK.

Checked Shuttle board SKP-412 LED feedback and table on MMI error code. It seems, software and hardware is OK but rather the feedback to shuttle board is N.G.

Checked Shuttle board connection for Shuttle 1 and 2.

Found Shuttle 1 connector, swapped. (CNS1-SLT and CNS1-D1 connector)

Revert to correct connection. Checked timing sensor and gate sensor alignment, cycle run from initial start and continuous start mode, no more error.

Side sensor used however, J502 persists. Due to time constraint, handler to release in side sensor not used as agreed by customer as their change kit has through hole.

Cycle run 7x7 and 4x4, able to plug and play without further adjustment of timing and gate sensors.


 


 

Cause

Wrong connection for Shuttle 1 connectors on Shuttle Board; Found Shuttle 1 connector, swapped. (CNS1-SLT and CNS1-D1 connector).
