Service History

Subject: NS-7080W TAC_Try variable overflow = 100

Handler Model: NS-7080W(NS7KW-08)

Controller: RC520

Date: 30 Aug 2010


 

Symptom

Feedback from line tech last nite that for NS-08, when operator enabled all 10 slots of Automat (Cat 3 – 3 slots, Cat 4 – 3 slots, Cat 5 – 3 slots and unknown – 1 slot), handler having hangup (TAC_Try variable overflow = 100) error.


 

 Action taken: Power OFF/ON handler, same. Automat over travel, manually move automat to safe position and avoid using all the slot. (Only use slot 1,3,5 & 10) OK. Under monitoring now.


 


 

Action

If it happens again, can you ask them to get a screen shot of the error message? Because we need to know it happens on which manipulator(robot). Last time, it happened to NS7KW-01 before and was for the output arm.


 

There's 2 scenerio this error may happen.


 

Scenerio 1

1) After Bin assignment change, handler must start from INITIAL START mode.

2) This is a protection feature to prevent mis-sorting in the event bin assignment change and CONTINUOUS START mode.

3) Thus, this error message may appear when controller does not understand which output tray binning to use in continuous start mode. Remedy for this case is usually just change to initial start mode and press START.

 

Scenerio 2

1) Error happens in the middle of production run.

2) This may be a different issue from scenerio 1 but we need to know which manipulator(robot) is causing it and what position when it happens.


 

If possible, kindly check with operator:

1) Did it happen during start or middle of production?

2) If it happened during start, was it after bin assignment change? If yes, was CONTINUOUS START selected or INITIAL START mode?

3) If it happened middle of lot, was automat at any particular slot? Was tray already out to unloading lane?


 

Have checked with the nite shift techs and error happened during scenario 1, during start of production, bin assignment change & CONTINUOUS START selected.


 

In this case, please disseminate this information:

1) When ever bin assignment is changed, please ensure INITIAL START is selected.

 EPSON described this error as: The controller detected the bin assignment changed and CONTINUOUS START selected, to prevent a mis-sorting, thus issued the error message.
