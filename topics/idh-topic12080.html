Service History

Subject:  STM NS8080-02  Plunger  broken issue

Handler Model: ns-8080 (NS8080-02, s/n:181581)

Controller: RC520

Date: 22~26 Mar 2010


 

Symptom

3x3 device was running on multitest handler.
 Checked the floating fiber amp setting, found that the input shuttle threshold setting has been changed to 470 instead of 600.
 Found the device pick up error at index arm might be caused the plunger broken. When the error happen if operator doesn't make sure the input shuttle device sit properly or in this case, standing
 and press "retry + start" button it might cause the plunger broken.
 Convert to 3X3 device package, due to encounter Hight frequence ontimes up error, found the tested software with bug.


 

23~24 Mar

Production running with 3.6X3.6 device with disable 2 site on site B and D due to loadboard problem.

Found another root caused the plunger broken. The input shuttle floating fiber sensor not detected when the device does not sit properly on input
 shuttle. That is only happened on certain slanted angle.


 

Plunger broken issue only happened on 3x3 package. Last week production only run on 3.6x3.6 package.


 


 

Action

Recommend and changes on Handler parts

1)KA will change hard material for plunger to prevent broken again.

2)KA will re-design the input shuttle pocket to prevent some slanted device which unable detected .

3)Recommend Meilin to add cylinder pushes on tray loader to minimize the tolerance.


 


 

Cause

- floating device cannot be detected due to Fiber sensor threshold setting change to 470 instead of 600.

- 3x3  device slanted cannot be detected


 

11 Apr 2010

For the root caused of the broken plunger issue.

1)Too big of tolerance at tray loader position for small device 3X3, caused the intermittent floating error at input shuttle.

2)Inappropriate size of tolerance for the tray at loader due to high frequence floating error at input shuttle. The technician always changed the higher threshold instead of 600.

3)The input shuttle floating sensor not detected when the device does not sit properly on input shuttle.

It is only happened on certain slanted angle. For time by time the plunger pick up the slanted device at input shuttle, it caused the plunger surface was chipped off due to device pick up error or device drop error at index arm. Thus, it will caused the plunger broken.
