Subject: NS-6040 Restoring backup data after recovery Windows

Handler Model: NS-6040

Controller: SRC520

Date: 16 Jul 2012


 

Symptom

UTL found the NS 6040 m/c can not boot up  and replace the new hard disk and recovering Windows already but need help to advise about " How to Restoring the Backup data " to the new hard disk .

Please see data back up in attached file ( From the old hard disk ) in attached file and advise.


 


 

Action

For NS-6040 restore, main items very important:

1) SPEL system data – motor calibration data.

2) SPEL pose data – pick and place position data. For your case, it is in Project.

3) MMI Setup file – production setup for all package/device. If unable to restore, no choice but re-create all again.


 

Your backup is very old. 5 years ago. So if restore is unable to run, it means you need to do manual calibration, teaching and data input.

I’ll guide you first on normal restore:

1) Go into SPEL, switch to P(Program mode) and Click Abort all to close all programs running.

2) In SPEL, click Project –> Close

3) In SPEL, click Project –> Restore –> All Projects

4) In SPEL, click Project –> Restore –> System

5) In SPEL, click Project –> Open (you should see only the one folder eg NS6000. Or select the latest date)

6) In SPEL, Switch to A(Auto-Mode) and start background tasks.


 

You should at least be able to HOME handler and dry cycle.

If you still encounter some error like Torque error, it means your back up data is not good for this handler.

In this case, you’ll need to:

1) Manually input Drive Units data (4D/GAIN and 4B/OFFSET) into SPEL (Refer Maintenance Manual, Chapter 8.4.3 CT Data to Chapter 8.4.5 CT Data Setting)

2) Calibrate all robots in SPEL (Refer Maintenance Manual, Chapter 3 Replacement Of Motors)

3) Manual Teaching for all robots points in SPEL (Refer Teaching Manual)

4) Create setup file in MMI (refer User's Manual, Chapter 4.2 Setup File).

RUN and CONFIG folder is the settings of the handler data like security, tower light settings, tester settings etc...
