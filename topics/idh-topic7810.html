Service History

Subject: Wrong site mapping between Verigy tester and handler

Handler Model: NS-7080(Xilinx)

Controller: RC520

Date: 16~17 oct 2008


 

Symptom

Wrong site mapping between Verigy tester and handler at Xilinx.


 


 

Action

There is basically no problem with the communication between Verigy tester and handler. ADVAN type 1 is able to use with quad site and ADVAN8 is able to use on octal site and there is no mis-mapping when communicating between tester and handler.  After checking with Xilinx Engineer, found out that they used “dummy_hnd.exe” to try out the communication instead of using MMI.  Had already advised Xilinx not to use this file for communication check.


 


 

Cause


 


 

Remarks

For GPIB, "ADVAN type 1" is able to use up to quad site.

"ADVAN 8site" is able to use on octal site and above.
