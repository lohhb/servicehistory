Service History

Subject: SEP NS-6040 HDD Recovery

Handler Model: NS-6040 S/N: 15678 (NS6040-2)

Controller: SRC520 (S/N: 02087)

Date: 4~6 Jan 2012


 

Symptom

SEP NS-6040 HDD Recovery.

Mfg: Aug 2002

2.5" Fujitsu 20GB HDD

Win95(J)

SPEL95(J) 2.53i

MMI 1.4a-JPN

Pentium MMX 200MHz

SRC520CE S/N: 02087 (MT355, CU-M313)

DU 1 S/N: 02947,     DU2 S/N: 02948,     DU3 S/N: 02949


 


 

Action

4 Jan 2012

HDD Recovery for English Win95, SPEL95 Ver 2.53i & MMI Ver 2.0c1

Restore SPEL Pos & System and setup files provided by customer.

Calibration of all robot axes(robot 1~6) by MCORG in SPEL95(E).

Encountered handler dry run with IFUNT100 error due to unable to open QAsample.def.

Also encountered runtime error #75.

It could be that backup files in japanese unable to restore to working condition in the english MMI & SPEL.

HDD Recovery to Japanese language from another NS-6040.

Restore SPEL Pos & System files from another backup.

Calibration of all robot axes(robot 1~6) by MCORG in SPEL95(J).

Dry run handler, found pick and place position out.

In SPEL teach position(pos) data for Index Arm(Robot 1), Shuttle 1 (Robot 5)   & Shuttle 2 (Robot 6).


 

5 Jan 2012

Empty Tray Arm(robot 4) unable to pick up empty tray as Empty Tray Arm not at centre of tray.Some trays, due to design of the tray, the tray pickup area for vacuum suction may not be at the centre of tray.In [Tray form], usually, "Pick Up Position" is 67.90mm.So may need to change this  "Pick Up Position" value if the Empty Tray pad cannot pick up at tray's centre.

Teach SPEL position(pos) for Empty Tray Arm(robot 4)

Running check with dummy devices, found Input Arm, Shuttles & Output pick & place position out

Encountered Index Arm unable to pick device during AutoContact Height.

Reteach Index Arm Z & U height (about -101mm)

Index Arm AutoContact Height, able to pickup devices.

Running check with dummy devices, found all pick & place positions out by at least 3mm when MMI offset is 0.

Suspect SPEL robots position(pos) data in backup are wrong.

During running, after release devices to Output Shuttles, devices fly off to outside of shuttle’s pocket.

Adjustment of Index purge air volume.


 

6 Jan 2012

Use NS-6000 teaching jig to teach SPEL positions(pos) data for:

- Input Arm(robot 2)

- Output Arm(robot 3)

Use setup file VFB10-144-02-02.

Index Arm AutoContact Height, able to pickup devices.

Run dummy device found pickup from loader drift.

Check Tray Matrix in MMI, found it does not match with current tray.

Key in new Tray Matrix.

Running check with dummy devices, pick and place offset alignment in MMI.

During running, after release devices to Output Shuttles, devices fly off to outside of shuttle’s pocket.

Adjustment of Index Counter-Air on time to 0.5sec and adjustment of Index purge air volume.

Reload IFUNT100 driver using Loader.exe.

Release handler to customer.


 


 

Cause


 


 

Remarks

12 Jan 2012, after setup package 1-QFP05-100-3-2-01. – Output Arm error code T- 4001 "The Arm reached the limit of motion range" during cycling and running mode.

Usually, after HDD Recovery if Input or Output Hands encountered error 4001 "The Arm reached the limit of motion range", it is advisable to make sure that Input or Output Arm Z & U in mnp02.pos or mnp03.pos are not zero, usually Z & U are -20 to -25mm.

Use GenPos in SPEL to teach the Z & U height for Input Hands height teaching with jig place on Input Shuttle and for Output Hands height teaching with jig place on Output Shuttle.


 

For Robot 2 Input Arm, make sure Z & U are not zero, around -20 to -25mm for the following points:

P50  ' Load Tray

P51  'Input Shuttle1

P52  'Input Shuttle2

P53  'HotPlate1

P54  'HotPlate2


 

For Robot 3 Output Arm, make sure Z & U are not zero, around 20 to -25mm  for the following points:

P81      'Auto1

P82      'Auto2

P83      'Auto3

P84      'Fix1

P85      'Fix2

P94      'Fix3

P95      'AutoMat

P96      'Fix4

P86      'Output Shuttle1

P87      'Output Shuttle2


 


 

Refer 0097-NS6K and 0103-NS6K.


 

For flying device after release from Index Arm to Output Shuttle, check points:

1) Redo Auto Contact Height (after AutoContact Height increase the device release height by 0.5mm to 3mm)

2) Index Nest (worn out? such as chip off?)

3) O-ring worn out on plunger (harden? broken?)

4) Plunger in the contactor(dirty? plunger able to spring out?)

5) Spring in Contactor (is it dirty? in good condition?)

6) Rubber cup in Contactor (sticky? that stick device? harden? worn out?)

7) Index Counter Air On Time (increase or decrease?)

8) For 1x2 or inline4, Output Shuttle Device Detection sensor(horizontal) alignment & working condition

9) Index Counter Air (purge air) volume eg. is it too strong or weak
