Service History

Subject: STM NS-8080 Octal site double devices sensor upgrade issue

Handler Model: NS-8080

Controller: RC520

Date: 23 May 2011


 

Symptom

-STM NS8080 Octal site double unit sensor upgrade

EP-03 s/n:181868

-Octal site double unit sensor bracket installation.
 -Fiber sensor installation.
 -HIM upgrade to Ver10.20MSM.
 -Spel and HMI IO lable changed for docking sensor.

 


 

Action

1)Received the OMRON digital fiber sensor E3X-MDA Series.
 found that the sensor amplifier wire has only 2 wires with one output and one 0V. It is supposed to come with 4 wires with 2 output and one 24V and one 0V.


 

2)Encountered high frequents of detection errors at output shuttle.

-Decreased index arm counter air to 0.1sec but error still persist.

-Detection errors solved after decreased release height by 0.5mm.


 

3)Found the output shuttle 1 sensor amp display have been changed to analog.

-Digital display solved after Initialize sensor amp to default setting.


 

4) using GP-IB tester interface Advance type 1 - 4 sockets passed and 4 sockets failed with “ERROR” Bin.
-After changed to Epson type M, the “ERROR” Bin was solved.
 -Units passed on socket hand test but found high frequents open bin 6 during docking with tester.

-Found socket sensor bracket touched with load board and it caused the open bin.

-Open bin solved after isolated the socket sensor bracket which touched the load board.

-I will swap back the existing socket sensor bracket by Monday.

Octal site socket sensor upgrades EP-03.

-Open bin solved after swapped back to the existing socket sensor bracket. (I think is because of the docking is not co-planar).

-Checked socket sensor functionality. - passed.

-Encounter GPIB category errors during re-test rejected units with GPIB tester interface with Epson type M.

-GPIB category error solved after Re-installed tester driver.

-Currently GPIB tester interface using Advance type 1.


 


 

Quad site EP-02

-Customer feedback that they encountered high frequents error at input arm “hand-up sensor doesn’t turn on”. (Hand D)

-Hand height calibration for input arm but error still persists.

-Error solved after changed the hand D hofs value.
