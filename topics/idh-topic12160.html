Service History

Subject: AMD(S) – UBS09 (S/N: 151044) IFUNT200 installation

Handler Model: NS-UBS

Controller: RC520

Date: 16 May 2013


 

Symptom

AMD(S) – UBS09 (S/N: 151044) IFUNT200 installation.MMI A13.22-SACRC520 S/N: 01-20500DUBox010: 250352 NFSPI-6941 with EasyRestore


 


 

Action

Contec PO-64L (for 7 Segment LED) is located on PCI4 slot in RC520.Insert IFUNT200 card to PCI2 slot of RC520.Power up handler, previously IFUNT card driver installed by customer.MMI & SPEL aborted, launch “C:\PSPELWORK\EXE\HWConfE.exe”.SPEL Input I/O board, add/change address for IFUNT to E800h and E807h.SPEL Output I/O board address, add/change address to E804h.Device Manager check Contec PO-64L address (E000h) and IFUNT100 address (E800h).For SPEL Input, the [Number of ports] has to change to 20 for "Uniwire" after install IFUNT card.
Launch “C:\PSPELWORK\EXE\Loader.exe” to load Hex file “IF100_41.Hex” and Verify, OK.IFUNT200 Interface for GPIB LED able to light up when GPIB was selected.Customer will use dummy tester to verify the GPIB communication.SR. 013457


 

Cause


 


 

Remarks

UBS handler software A13.23-SAC with GPIB “GETBARCODENUMBER” not working. EPSON dispatched software designer Maeda-san to rectified it.
