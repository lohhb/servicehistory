Service History

Subject: NS7080W SRC error on empty tray arm and index arm.

Handler Model: NS-7080W (STATS ChipPac Ltd,S/N: 121281)

Controller: RC520

Date: 13~15 May 2008


 

Symptom

SRC error on Empty tray arm and Index arm.

Tracing through the event log, found out that the SRC error might due to tray pick up error.  The scenario (from event log) was that there is tray pick up error, then the operator pause the handler and open the side door and after that they started the handler and SRC error occurred. They might have moved the empty tray arm when they opened the side door to correct the tray pick up error.


 

Action

After confirming that there was no abnormality (e.g. bent, dent and etc) on the empty tray arm, pressed “HOME”, and saw that empty tray arm hit on the hard stop and SRC error came on.  Did a “OFF”, ”ON” on the handler and tried “HOME” again and the same thing happened. 2 errors occurred, they were namely, error 5040 – Torque error in the High-power state and error 5041- Torque error in the Low-power state.  These 2 errors are common when SRC error occurred.

Use POS data from backup, still hit hard stop.

Use “J/T” in SPEL CT to move empty tray arm to P44, still got the error.

Teaching carried out, but still hit hard stop.


 

Following checks carried out:-

- if the arm moves around smoothly when it is moved around by hands,

- if the arm's movement is tight and secure when it is moved around by hands,

- is there is any hitches with the arm when it is moved around by hands,

- if the belt is slackened,

- check if the manipulator's arm is not colliding with the peripheral units around, and

- check on the power cables inside the manipulator for any loose pins, cable disconnection or short-circuit.


 

Did an encoder reset and calibration of ABS motor and problem solved.


 

Finally, “home” pressed and empty tray arm can home with no problem,

but index arms both cannot home and both arms instead of move up, they moved all the way down and a 'bang' sound was heard.  There is no SRC error at all.  Suspect 0 pulse position for both index arms were out, index arm encoder was reseted and abs motor calibrated.  Did “home” again, everything was good.  Tried a 2nd times, both the index arms cannot home and extended fully downwards.  AGAIN reset encoder and calibrated the motor, everything was good again. Tried a few times Homing, everything was good.

Set up 20x20 package and run 7 trays of dummies, everything was good.


 


 

Cause

Empty arm crashed and cause SPEL point out of position.

Empty tray arm motor encoder reset and calibration solved the problem.
