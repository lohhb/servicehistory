Cause

The GPIB communication error that happened on 19 Dec for NS-101(S/N: 151243) was found due to the following reasons:

1) old HMI software C8.10-SIK (FD08S025) pre-installed in EPSON NS-8040 both sub-con makers.

2) EPSON-M loaded IF100B17.HEX for GPIB driver(by Loader.exe).

EPSON-Z loaded IF100B13.HEX.

Why double standard, not standardized. WHY?

I cannot confirm whether IF100B13.HEX also can be use.

Please inform EPSON-Z for immediate action without delay to make this correction.

 

3) Product ID for handler is NS8040 for EPSON-Z.

Product ID for handler is NS-8040 for EPSON-M, this is correct as per Infineon requirement.

If V93k tester read Product ID as NS8040 will give error or HANG UP.

Please inform EPSON-Z for immediate action without delay to make this correction to NS-8040  .


 

NS-101 itself is not equipped with SPEL special commands

(eg. ID? , :TEMP:SETPOINT? , :TEMP:STATUS? , :SITE:MAPPING? ) so reload the correct HMI CD for SPEL, and reload HEX 17.


 

Remarks

Refer email 26 Dec 2009

This "GPIB connection error" could have been avoided if :

1) HMI F8.12-SIS was pre-installed in EPSON sub-con factories when making the NS-8040

2) Correct Product ID was used (NS-8040)

3) Latest GPIB driver IF100B17.HEX was loaded

TESTER_COMM.PRG 44KB 10/22/2008 11:56AM(Modified) is correct

By installing F8.12-SIS, the SPEL CT project will be copied into the SPEL CT 3.11ds  including the 4 GPIB commands.


 

Mr. KT Goh mentioned in 16 Oct 2008 email, the commands type in at the tester web page are:


 

ID?%0d

:TEMP:STATUS?%0d

:TEMP:SETPOINT?%0d

:SITES:MAPPING?%0d


 

Effectively, as the web software will later append a line feed, the handler will receive <command>CRLF


 

In J750 and CX,  the National Instrument ibic software doesn't appendany terminator.

A command must be followed by terminators \r\n, the \r

and \n being denotation for the ibic to translate to a CR and LF

respectively.
