Service History

Subject: NS-7080W hang during Auto Run at Hot mode 90DegC, MMI status show "Pause". In SPEL CT error history shows "Unknown error!", Function AM_IDX_IF. Index Arm 1 at socket

Handler Model: NS-7080W(S/N: 121040; NS7KW-06)

Controller: RC520 (S/N: 01-20283)

Date: 2 Sep 2010


 

Symptom

Handler suddenly hang during Auto Run at Hot mode 90DegC, MMI status show Pause. In SPEL CT error history shows "Unknown error!", Function AM_IDX_IF. Index Arm 1 at socket.


 


 

Action

-Replaced to new 2.5" 40GB HDD

-Software Recovery by Recovery CD Ver 1.20 as stated in EPSON Factory certificate, but did not work

-Software Recovery by Recovery CD Ver 1.0 FD04S050 (EasyRestore)

-After restart Handler, Windows 2000 detect Multifunction adapter for CONTEC PO-64L(PCI) - for Automat 7Segment LED display, NStask bar loading but hang, mouse cannot move

-Force restart handler, press F8 and go to Safe Mode for Windows 2000

-In safe mode, remove NS-task short cut from Program->Start

-Restart Handler

-launch C:\PSPELWORK\exe\HWConfe.exe, check Windows 2000 Device Resources for IFUNT100(Input/Output Range lower address), CONTEC Co.,Ltd.-PIO-32/32L(PCI) & CONTEC PO-64L(PCI) address to same as in SPEL I/O Board respective address

-Add SPEL I/O Output card for Contec 64L card

-Re-install SPEL CT to 2.81b1H

-Upgrade MMI to 8.10-SXC

-Restore SPEL System data, Pose data and MMI setup files

-Cycle run handler at 90DegC

-Shuttle.res error when click Shuttle in UnitSet

-Copy Shuttle.res from NS7KW-08(in C:\NS7000\Localize\Env\English) and overwrite in this handler, ok


 


 

Cause

Faulty 2.5" HDD.


 


 

Remarks

2.5" HDD without OS, R13A110401000


 

Automat installed in August 2010

RC520 S/N: 01-20283

DUBOX010 S/N: 250203NF

CPU Board SPI-6941, 3 PG Board(C-864v1) on RC520 Rear-Left

Contec Co.,Ltd.-PIO-32/32L(PCI)

Contec Co.,Ltd.-PIO-64L(PCI)

MMI 8.10-SXC   SPEL CT 2.81b1H

40GB HDD

Cannot use Recovery CD Ver 1.2 : SPI-6941 + PG Board C-873 +Win2k

Use Recovery EasyRestore CD 1.0 : SPI-6941  + PG Board C-864v1+Win2k

Maybe type of PG Board used will also affect which Recovery CD version to use.
