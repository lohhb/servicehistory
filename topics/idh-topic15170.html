Service History

Subject: NS8080 8 hand Encounted the “no information”errors during production

Handler Model: NS-8080 (SN:181655 , NS88-04) 8 Hands

Controller: RC520

Date: 21~22 Feb 2011


 

Symptom

ASE 2 – NS8080 8 hand Encountered the “no information”errors during production.


 


 

Action

21 Feb 2011

-Customer feedback that they encountered the “no information”errors at input & output arm during production.

-Upgrade HMI to Version 1.8

-Upgrade Spel to Version Version 3.11dh-sp

-After “Protect test site Assign” option was checked, found “Handling mode” unable to select.(wrong)

-Downgrade to HMI version 1.3, “Handling mode” able to select(correct)

-Physically check all actuators conditions found input arm hand B o-ring missing.

-Running check for 3 hours without any ”No information “errors found.

 

22 Feb 2011

-Running check with units for 1k without any ”No information “errors found.

-Meeting with kate leong and hidaka san

below is the items that we discussed on the NS8080 8-hand handler issue.

Software Ver 1.8 issue (this version resolved the “No Information Error” from input/output arm) This version of HMI does not allow user to select “handling Mode” in the “Setup” if the “Protect Test Site Assign” is being turn on.

ASE II needs the handling mode to be selectable with the “Protect Test Site Assign” on, similar to the current HMI Ver 1.3.

Epson to check and make modification on Ver 1.8 HMI, or newer version of HMI if available, for ASE to upgrade.

Epson to look into the way of managing the “Standard Handling Mode” in new HMI release, so that ASE and can continue to use the future new release HMI without having problem stated in “a”.


 

Jamming of input/output Z hand recovery method

Current recovery procedure requires to “Reset” handler in order to clear the jamming if encounter Z hand up/down jamming.

This may impose a quality issue if a DUT still under testing if handler being reset before DUT finish testing.


 

-Propose to change the recovery method that just to reset input or output arm only, so that the handler does not need to be reset.
