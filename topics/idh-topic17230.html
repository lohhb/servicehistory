Service History

Subject: NS-8080HW Contact test alternate Index Arm fail

Handler Model: NS-8080 (S/N: 182025)

Controller: RC520 (S/N: 03-2061)

Date: 23 Nov 2011


 

Symptom

NS-8080HW -Manual contact test in OneCycle mode. Arm 1 Contact test passed, Arm 2 Contact test test failed, Arm 1 Contact test contact test failed.Alternate Arm Contact passed.Current HMI T10.56-KSS.


 


 

Action

SMC valve reading for Arm1 was 0.427 Kpa during High Contact Force, Index Arm 2 was 0.137 Kpa-Using Index Arm1 (Disable Index Arm 2), Handler manual contact test in Ambient but unable to communicate with tester, found handler's driver not loaded in tester. Tester Engineer load NS-6000 driver to tester, GPIB able to communicate with address 6.-Ambient functional tester with Handler but not all function tested as test program incomplete.


 

-Enable Index Arm2 only, manual contact test in One Cycle mode.

First Contact test passed,

 2nd contact test failed,

3rd contact test failed.

 Alternate pass cycle, according to Hidaka-san this could be a HMI software bug.

Advise to install HMI T11.52-KSS.


 

24 Nov 2011

After installed HMI T11.52-KSS, ok.


 


 

Cause

Software bug in T10.56-KSS.


 


 

Remarks

Regarding for contact force on the SMC display, it shows transfer contact force.

(The transfer contact force is while Index hand pick up devices at Input shuttle.)

I think that the latest software should be kind of stable number on the display.


 

According to Hidaka-san email on 12Dec 2011:

Ver11.52-KSS (based on HMI 3.2)  do NOT have 6 bins support. Because, it is special software of Korea.

After Ver3.6 software is supported 6 bins(Auto 1, Auto 2, Auto 3, Fix 1, Fix 2, Fix 3).


 


 

T13.70-KSS (Released on 15 Mar 2013):

1. Fixed a Bug when Online Bin enable Double Contact Motion in High Contact Mode (Online test Error 4024 for Index after test).

2. Fixed a Bug that Output Arm crash into Tray Arm during Tray Changing (Found in ASE NS-8080 8 Hand crashed 14 Mar 2013).
