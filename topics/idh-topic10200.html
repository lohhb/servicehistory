Action

Does customer have any special software/hardware special options? Like scanner etc..

 

Let's look at T-4216, T-4233 and G7101.

For the G7101, I believe there should be some error message displayed on screen. If possible, can you ask customer to screenshot this error so that we can confirm if it could be just a registry patch. I believe this error message which pop out on screen was by windows/OS and not by SPEL/HMI. That was why our error history shows "Unknwn Error" but I believe there's some message shown. If it's just a registry patch, EPSON still need to know what was the message so screenshot it if possible.

 

Nevertheless, all 3 errors, it seems related to some synchronizing of data error/timeout  (T-4216/T-4233) or CPU too busy (G7101). It's like some delay in data transfer/receive and timeout. How does customer recover from each of this errors? Control Panel "Power OFF/ON"? Able to check the DU led was it a resettable or not resettable error?

 

Checkpoints:

1) Cable connection between control unit and peripheral units. Don't assume they're good, rather remove and check the pins etc..

2) Connectors mainly EMERGENCY, DI/O, RI/O. Check on the short-circuits with the connecting cables or/and inside the peripherals unit.

3) As it seems to be affecting all manipulators, highly suspect not pertaining to single MIB or single DU slot etc. Better to swap Control Unit but you may not have the luxury of a CU. In this case, you may have to swap CPU Board, ram slots (or resit), increase rams if customer has many optional peripherals/interfacing/comms etc.

 

If the physical checks on connectors are good, kindly use a new HDD and recovery CD to recover to base version with system/pose data restore. You mentioned you checked the intergiry but it is only pertaining to bad sectors or file system error. The error checks showing no problem with HDD but an old HDD or a failing HDD for example may still have real time error. So perhaps in terms of effort and time, you may want to consider using a new HDD. Or it may just be software issue...
