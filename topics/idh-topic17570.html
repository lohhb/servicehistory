Service History

Subject: NS-7080 Arm 1/ Arm 2 device drop false alarm

Handler Model: NS-7080W

Controller: RC520

Date: 9 Jan 2013


 

Symptom

9 Jan 2013 - ASE(Penang) NS-7080 Arm 1/ Arm 2 device drop false alarm.

 


 

Action

1)Cycle run handler with units.
2) Witness the “Arm 1/ Arm 2 device drop false alarm”
3) Upgraded MMI to 2.0p
4) Continue cycle run handler, same problem occurred
5) Check/ fine-tune vacuum sensors’sensitivity with device. -
6) Found Arm 2 layout kit site A silicon tubing too long causing twisting, trim the tubing length. – Rectified
7) Continue cycle run handler, same problem occurred.
8) Check INDEX IN1 176 Input 8CH IO module addressing. – Correct
9) Check INDEX IN1 176 Input 8CH IO module light indication corresponding to respective sites on the index arm by triggering the vacuum from MMI. – Correct.
10) Request from Kamijo san to check index arm vacuum check wait time. – Regardless of the how long/ short of the check wait time, problem still occurred.
11) Found false device drop error at Arm 2 happened during initial start during index arm checking on test site.
12) Found false alarm occurred on arm 1 or 2 when using SINGLE shuttle 1 or 2.
13) Request from Kamijo san to check and take photo of the back/ inside of controller.
14) Put back controller and power up handler, found handler unable to boot up to windows.
15) Unable boot up handler even in safe mode.
16) Request recovery CD from Raj and perform HDD recovery using spare HDD from Chuah.
17) Able to boot to recovery CD but unable to boot into HDD for windows installation.
18) Informed Raj about CPU Board unable to boot up and Controller’s PSU fan not turning.
19) According to Chuah, he also encountered handler unable to boot up to windows before.

1) Unable to boot up to windows - Request customer to swop or replace SPI-6941 CPU Board.
2) CPU fan not turning - Request customer to replace.
3) KGS propose ASE to send back handler back to EPSON for repair.
