Service History

Subject: NS-8040 “The Arm reached the limit of XY motion range” error occurred during homing,  “The Arm reached the limit of motion range” happened during running check.

Handler Model: NS-8040 (S/N: 182184 )

Controller: RC520 (S/N: 03-30076)

Date: 13 Dec 2011


 

Symptom

SCS NS-8040 “The Arm reached the limit of XY motion range” error occurred during homing, “The Arm reached the limit of motion range” happened during running check.

This controller S/N: 03-30076 is a black colour controller.


 


 

Action

- Recovery of HDD carried out using Recovery CD Ver 4.0 (FD11S057, 04/28/2011).
 - Latest version of HMI and SPEL installed.
- Restore pose data but unable to restore.
- Try to copy and paste the pose data but the HMI unable to launch.
- Reboot system still the same, so recovery again.
- After recovery, restore system followed by pose data. Everything look good.
- “The Arm reached the limit of XY motion range” error occurred during homing. So carried out calibration of robots. Thereafter, homing completed.
- “The Arm reached the limit of motion range” happened during running check.
- Compared the robot parameters and manipulators with the other system and changed some value but still persist and another error “communication error with temperature controller” happened.
- Swapped another system HDD over to try and the system did not shows “communication error with temperature controller” error and no error in homing.
- Considered to clone the HDD and do restoration but customer cloning software does not work on NS8040.
- Tried recovery and restore using another system recovery CD, HMI CD and backup. Everything looking fine.
- Restored the original System and pose data and calibration carried out on all robots.
- System able to run but pickup position out by about one and a half pocket.
- Compared the points with another system found not much different.
- Tried to teach the points using auto teaching jig but unsuccessful.


 

Cause

Bug in Recovery CD Ver 4.0 (FD11S057, 04/28/2011).


 


 

Remarks

Recovery CD Ver 4.0 (FD11S057, 04/28/2011) for NS-8000 4 hands has a bug.

NS8000 Handler which using the new controller (Black) RC520 S/N: 03-3**** has problem when doing recovery using Recovery CD Ver 4.0 (04/28/2011) FD11S057 for SPI-IB945 CPU Board.

After recovery, “The arm reached the limit of motion range” error happened at input/output arms.

Only POSE and SYSTEM data were restored at the point of recovery.

Tried open up the range for both input/output arms but result still the same.

After recovery, the Hand Pitch at Handler Configuration for X-pitch was at 40mm. these pitch couldn’t be changed even HMI was reinstalled.

The only way to changed was to through the handler.def from the config file and the X-pitch value should be 80mm.

or copy back the Config folder.


 

Use Recovery CD Ver 4.1 (FD12S019, 02/13/2012) for 4 hands to solve the problem.


 

Refer 0443-NS8K for same issue.
