Service History

Subject: UTAC NS-8080 Device condition error on Output shuttles

Handler Model: NS-8080(S/N: 181665, NS8080-01) 8 Hands High UPH

Controller: RC520

Date: 15 Dec 2009


 

Symptom

Device running on this octal setup is BCM2070C_4.5x4

Feedback from the line, handler encountered many device condition error on output shuttle 1 and 2(device floating in pocket after test).


 


 

Action

Tues 15/12/09

- Feedback from the line, handler encountered many device condition error on output shuttle 1 and 2(device floating in pocket after test).

- Before we checked handler, line tech already lowered index release height of arm 1 and 2 to 1.00mm above pick up height and jams reduced.

- Monitored and encountered device condition error at output shuttle 1 pocket C intermittently.

- Further lowered index release height to 0.5mm above pick up height.

- Did not see jamming on output shuttle 2 so we swapped both output shuttles. To see if problem migrate.

- After running sometime, started to see problem at output shuttle 2 pocket C. Device float inside pocket after test.

 

Wed 16/12/09

- Checked jam history and found many device condition error at output shuttle 2 started around 9am. Night shift only encountered 1 case.

- Swapped output shuttle 2 and input shuttle 2.

- Continue to have device condition error on output shuttle 2 but this time device did not float in C pocket. Instead, it sticked onto the index arm 2 site A contactor and the shuttle site A missing device in pocket.

- Swapping of arm 2 contactor by line tech.

- Still jam.

- Complained by line, handler unable to meet their index time of 0.6s. They were getting 0.62s and above. Increase handler running speed from 90% to 100%. Index time became 0.58s.

- Increase arm 2 release height back to 1.00mm above pick up height. Continue to have jam intermittently.

- Increase arm 2 realease height to 1.20mm above pick up height.

- Lot ended. Continue monitoring tomorrow.


 


 


 

Cause


 

Remarks

13 Jan 2010

Checked device condition error in EEMS for NS-8080 8 Hands High UPH handler.

Contact head design was same with UTAC new version.

However EEMS is using Semitron plunger instead of metal.

I will request to KA to share with us the drawing of contactor heads


 

For Output Device Condition error, check points:

1) Redo Auto Contact Height (after AutoContact Height increase the device release height by 0.5mm to 3mm)

2) Index Counter Air On Time (increase or decrease?)

3) Remove Output Shuttle jig, check below the output shuttle plastic if it is dirty or have device

4) For Square site(eg. Quad, Octal..etc), check Gate Sensor(eg Receiver) is working properly and it's position

5) Output Shuttle Sensor Board (SKP456C, SKP456D, SKP456E)

6) Gate Sensor Board (Shuttle board)

7) For 1x2 or inline4, Output Shuttle Device Detection sensor(horizontal) alignment & working condition

8) Index Nest (worn out? such as chip off?)

9) O-ring worn out on plunger (harden? broken?)

10) Index Counter Air (purge air) volume eg. is it too strong or weak

11) Plunger in the contactor(dirty? plunger able to spring out?)

12) Spring in Contactor (is it dirty? in good condition?)

13) Rubber cup in Contactor (sticky? that stick device? harden? worn out?)

14) Wire connection on sensor(eg. broken wire, connector connected properly to sensor board?)

15) Input I/O Module for EIN10 (SHUTTLE3) / EIN13 (96) near to Gate Sensor Board(Shuttle Board)

EIN 10 16CH for NS-8020, NS-8040

EIN 13  8 CH for Timing sensor IN148, IN149 (For NS-8080, NS-8160)
