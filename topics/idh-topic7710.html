Service History

Subject: Upon “HOME” button, no reaction from handler although MMI shows “RUNNING”

Handler Model: NS-6040(CARSEM)

Controller: SRC520

Date: 3~4 Sept 2008


 

Symptom

Upon “HOME” button, no reaction from handler although MMI shows “RUNNING”.


 


 

Action

1.   Replaced CPU Board and Z motor.

2.   Upon “HOME” button, no reaction from handler although MMI shows “RUNNING”

3.   Unable to motor on in SPEL. Command executed but no motor engagement

4.   Motor status shows motor on and low power.

5.   Checked IRQ5 and IRQ10. OK

6.   Checked Contec IO board selector switch, position 0.

7.   Cabling and connection all confirmed by customer as ok.

8.   “PG Board communication error” after press “HOME” button but no movement and MMI showing “RUNNING”.

9.   Advise customer to separate between H/W or S/W issue.

10. Using another handler HDD to verify. In a case of S/W, possible SPEL system files/IO addressing/Drivers corruption.

11. In case of H/W, to check for torque upon “POWER ON' of handler.

12. As long as I/O pickup hand unable to “HOME”, sequence will not complete and hang up as above.

13. In case of unable to “MOTOR ON” all robots, to check for general source like DU/CU.

14. Customer will try and feedback.


 


 

Cause


 

Remarks

- Can try delete SPEL project then re-install MMI which also install back the SPEL project.

Backup project before delete it.

(Project->Close , Project->Delete)

- Check front operation panel
