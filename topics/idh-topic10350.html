Service History

Subject: NS-6040 CPU Board downgrade from CPU BoardPC-686BX (Windows 2000) to P5/6X86 CPU Board (Windows98) issues

Handler Model: NS-6040

Controller: SRC520

Date: 28 May ~ 12 Jun 2009


 

Symptom

NS-6040 CPU Board downgrade from CPU BoardPC-686BX (Windows 2000) to P5/6X86 CPU Board (Windows98) issues relating to backplane board.

Keyboard unable to detect after downgrade.


 


 

Action

11 Jun 2009

Take CPU board PC-686BX(PC)-EEP850 from NS60-35T(S/N:15904) put into NS60-54 s/n:121024 which they want to downgrade from Win2k to Win98 but encounter Keyboard unable to detect error.


 

We found both controller have different backplane board.

NS60-35T which is running Win98 has an older backplane board, while NS60-54 which is running Win2k has a newer board and controller is RC520 type.


 

Can you advise whether CPU board PC-686BX(PC)-EEP850 can be install onto the newer backplane board?

PC-686BX can be used newer backplane board.

PC-686BX has keyboard plug on the back side.

You can connect keyboard plug to back side of CPU board directly.

The other idea: Keyboard for USB type

You should check PC condition.

It is available to set USB legacy support on the computer setup.


 

14 Jun 2009

The CPU board which cannot detect keyboard is P5/6X86. This is the earlier cpu type used on NS60-02.

CPU PC-686BX from NS60-35T can be used on NS60-54 on Win98 platform.

We have tried and it worked. However, we have not try CPU board SPI-6941 from NS60-54 on NS60-35T yet.


 

Tried cpu board PC-686BX from NS60-35T on NS60-54 and it can boot up with Win98 OS, keyboard can also be used


 

Refer email 28 May 2009 for P5/6X86 CPU Board connections and jumpers' setting.


 

15 Jun 2009 Pre-check condition or returning NS60-35T

Recovered system using Win2k recovery cd with cpu board SPI-6941 from NS60-54 but encountered hang up/no response from operation panel after restoration of system data.

As discussed with Iwata san, handler will be brought back to Getech and try to recover with teaching, calibration, set up data etc.

Also found vacuum pump faulty, thus unable to verify index arm and tray arm vacuum units condition.

Remarks: Do not restore SPEL95 system data into a CPU Upgrade(PC-686-BX or SPI-6941) system that using SPEL CT, this will cause hang up.


 

Cause

NS60-54 backplane board not compatible to use with P5/6X86 CPU Board.


 


 

Remarks

Old type Backplane

specification: MBP-PC114R (R35N600000100)


 

another old type Backplane is SRC520CE is MBP-PCI13L


 

New type Backplane

Spec:PCI-14S-RS-R30


 

Lastest Backplane is MB-PCI14/9(PC) which is also use in NS-7000.
