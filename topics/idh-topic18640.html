Service History

Subject: NX1032XS issues after HMI 1.3 installed

1) Timer stamp error for HMI 1.3

2) Devices placed outside of Fix2 & Fix3 tray pockets

3) Rotators cannot run above Speed 70%

4) Torque error in High Power State for Index Arm 2

Handler Model: NX1032XS (S/N: 182347)

Controller: RC520 (S/N: 00504), DUBOX030 (S/N: 100004), PSUNT353 (P892)

Date: 11 Oct 2012  (Recovery DVD Ver 1.0,  FD12S164)


 

Symptom

 NX1032 issues after HMI 1.3 installed

1) Timer stamp error for HMI 1.3

2) Devices placed outside of Fix2 & Fix3 tray pockets

3) Rotators cannot run above Speed 70%

4) Torque error in High Power State for Index Arm 2


 


 

Action

15 Oct:

- installed HMI 1.3 to solved Reloader 2 that cannot move out empty tray to operator's side, verified ok.

- time stamp error after HMI installed

- Devices placed outside of Fix2 & Fix3 tray pockets for some sequence


 

16 Oct:

- Input & Output Rotator stages can only run at 70% speed, if above 70% will only rotate device by 45Deg & not 90Deg.

- Ambient UPH for test time 0sec & 1sec done, but when set test time to 2sec, during first contact by Test Arm 1, Index Arm 2 tried to move towards Arm1 while Arm 1 still in test area.

This caused Error 5040 torque error in High Power State for Index.


 

17 Oct:

- HMI 1.3 beta1 installed, solved Fix3 & Fix2 device placement problem(tested at 75%, 90% &100% speed)

- time stamp error solved with new HMI

- PG parameter file pasted into C:\NX1000\Config\PG  folder solved rotation speed issue(tested at 75%, 90% &100% speed)

- HMI 1.3 beta2 received and installed to solve Error 5040 torque error in High Power State for Index when test time is 2 sec or more.

Verified with 2 sec to 8 sec seem ok with speed at 85%

- currently to monitor by running UPH
