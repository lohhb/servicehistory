Service History

Subject: MMI hang at spel executing or mouse and keyboard freeze(SRCLoad.dll error)

Handler Model: NS-6040(CARSEM, SEIKO3, S/N: 15887)

Controller: SRC520

Date: 13 Aug 2008


 

Symptom

NS6040 S/N:15887 , SEIKO03 at CARSEM – MMI hang at spel executing or mouse and keyboard freeze.

Uninstall the SPEL95.
 After re-installed the SPEL95, reboot the PC.
 Manually launch the SPEL95.
 Use Start -> Programs -> Spel95  and click spel95 icon.
 Prompt-up error as below:-
 "SRCload.Dll initialization routine error. Error Code: 0201"
 Press 'Ok'
 Come-out another error message as below:-
 "E-9201 SRCMng initialization error."

 


 

Action

Initially, replaced the suspected MIB2 with a new one but to no avail.  Thus individual boards in SRC520 from SEIKO04 was swapped to SEIKO03, it was found out that CPU board(PC-686BX) is the cause of SPELT/PC hanging.  We did a few times of verification check to affirm that CPU board is the problem, via putting SEIKO04 CPU board into SEIKO03 and vice versa.

Current status is awaiting for CPU board quotation.


 


 

Cause

Faulty CPU Board.


 


 

Remarks

For CPU board, in CMOS Bios->PnP/PCI Configuration->IRQ Resources-> IRQ-5 and IRQ-10 are set to Legacy ISA.
