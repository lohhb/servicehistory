Service History

Subject: Restored SPEL System data, encountered MMI status "Power down", unable to home or start handler.

Handler Model: NS-6040 (NS60-35, S/N: 15904)

Controller: SRC520

Date: 15 Jun 2009


 

Symptom

Requested by customer to try using SPI-6941 CPU Board in NS60-35 S/N:15904.


 


 

Action

15 Jun 2009

1) Backup setupfiles, system and projects from SPEL and backupsan of handler.

2) Converted CPU Board PC-686BX(PC)-EPC850 to SPI-6941.

3) Performed HDD recovery.

4) Restored OS Win2k, MMI 2.0a11 and SPEL 2.81b.

5) Restored SPEL System data, encountered MMI status "Power down", unable to home or start handler.


 

17 Jun 2009

1) Converted CPU Board PC-686BX(PC) to SPI-6941.

2) Manually restored MIB data(Motor drivers CT gain and offsets)

3) Restored POS data.

4) Calibrate/ MCAL all robots.

5) Restored setupfile.

6) Installed changekit and dry cycle run, OK.

7) Performed teaching on Index P70 and P71 as seen position out during Auto-height calibration.

8) Replaced faulty vacuum pump.

9) Check for vacuum units for index and empty tray arm, OK.


 


 

Cause

Found out that System data from SPEL 95 v2.53i is not compatible to use in SPEL CT v2.81b, therefore causing MMI status show "Power down".

The solution is:

1) in BIOS, set  [PnP/PCI Configurations] to Yes, Resources controlled By [Manual], Set IRQ 5 & IRQ 10 to Legacy ISA

2) Use EPSON CPU Upgrade Recovery CD to do recovery of Windows 2000, MMI and SPEL CT(CPU 850MHz)

3) Install PC I/O driver (Multimedia controller) (Contec Driver in Recovery CD or MMI CD)

4) Install IFUN100 driver (PCI Bridge) (Driver in Recovery CD or in MMI CD)

5) Use Loader.exe (in C:\NS6000\Exe) to load hex file (Hex file in Recovery CD)

6) Restore DU 1, 2 & 3 data in SPEL CT

7) Restore Robot Manipulator data in SPEL CT

8) Copy pos data files from backup project to SPEL CT project
