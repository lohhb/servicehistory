Service History

Subject: NS-8040 175DegC Handler device stuck in socket

Handler Model: NS-8040 175DegC (S/N: 151205)

Controller: RC520

Date: 6 Apr 2010


 

Symptom

NS-8040(NS-200, S/N: 151205) @160DegC PG-LGBA-208-4 17x17 evaluation support.

1st lot size is 1137 but only managed to test about 500pcs of device.
 Test time is about 30 sec at 160DegC.
 During the first hour test, no jam. After that there were jam for device
drop(stuck) in socket after test.
 JAM0303, Device drop error (Arm 1) : 12 times
 JAM0304, Device drop error (Arm 2): 6 times
 Most of the time, using vacuum pen to pick up the drop device(after test)
was impossible because the device were actually stuck in socket.
 - all 4 sockets were tightened properly but the external area that device
does not sit on can be compressed down, Infineon member said that this is
normal
 - all device drop/stuck in socket happened after test AND only happen on
test site C & D
 - have swapped socket B with D, and C with A but still of device stuck in
Test site C or D
 - have swapped the contactors for Arm 1 ; Head 1/2 with Head 3/4 but still
of device stuck in Test site C or D
 - unable to find any damaged to the nests for Head 1/2 & Head 3/4
 According to Vincent, last week when testing 60pcs of devices, during the last few cycles of test only Arm 1 device drop/stuck in test site D only
afew times.


 


 


 

Action

7 Apr 2010

Device drop in socket for Site C & D investigation offline, but offline run encountered only device drop in Site A

- Ambient mode, 3 trays of unmark dummy devices run for 2 cycles with dummy test time 30sec, ok, no jam at socket

- Hot mode 160DegC, 1 tray of production devices cycle run with dummy test time 30sec, 3 device drop at socket Site A all for Arm 2

-  Hot mode 160DegC, only enable Site A & using the 3 dropped device, cycle run and only 1 pc of device drop

-remove socket guide plate and loadboard, KA member check/measured at 4 sockets and production device, suspect socket issue due to wear & tear and also design issue.


 

Cause of reason (We guess)

1. Socket problem

2. Different height of socket base for right side and left side 3. Slant on test hand base plate 4. Insufficient the projection of test hand vacuum pad.


 

Countermeasure:

1. I already ask to socket supplier (Agent) and make the schedule to check condition.

  I'm waiting for their reply.

2. To adjust shim on the socket base.

3. Execute Height calibration during 160 degree

4. To adjust the value of projection of test hand vacuum pad  

   Current condition: Around 0.3mm

                      Change to 0.5mm (Adjusted by shim) 5. CK matter:

   Guide pin hole on the nest.

   One is for circle, the other one is for ellipse.

   Set the circle guide hole position to center side.


 

8 Apr 2010

Socket condition & devices checking with reference with NHK Japan's socket drawing with Hidaka-san.


 

9 Apr 2010

- File away part of the socket  to improve ball guide as requested by Hidaka-san, total 4 pcs

- Check Ambient running by 3 trays of Dummy device, ok.

- Check 160 degree running by 3 trays of Dummy device, ok

- Check 160 degree running by 1 tray of Engineering device, occur device pickup error on test site A.

- file some lines on the bottom of device guide to allow air to flow into socket to fill up vacuum during Index Arm pull up from socket.-

-Check 160 degree running by same tray of Engineering device, Disable test site C & D , no device drop in socket


 

27 May 2010

-Installed 4 new improved sockets from EPSON/NHK to loadboard.

-Ambient run check with devices that encountered stuck in sock, ok

-Hot mode set at 35DegC, running check with devices, ok

-Dummy test time set 30 sec at 160DegC:

1st try 1 tray, ok

2nd try 1 tray, ok

3rd try 1 tray, ok.
