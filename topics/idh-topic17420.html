Service History

Subject: NS-7000 T-4040 – communication error with the PG board(output arm)

Handler Model: NS-7000 (S/N: 18732, NS70-01)

Controller: RC520

Date: 20 Jan 2012


 

Symptom

Silab NS-7000 encountered T 4040 – communication error with the PG board(output arm).


 


 

Action

1. Checked SPEL error history. T-4001 – The arm reached the limit of motion range.

 Robot 8, axis 1 (This is Output Open/Close)
 2. Output arm run fixed ok, but when set to open/close mode, hand A and B will hang during auto run. Sometimes lost torque.
3.Checked output arm open/close pitch, 80x60mm ok.
4.Swapped input(DR20) stepper motor driver and output(DR21) stepper motor driver, output arm problem migrated to input arm. When input arm run 10% open/close speed ok, set higher speed will hang up during auto run.
5.Swapped back both drivers and revert MMI settings and cycle run.
6.Recommended part replacement: Stepping motor driver(DR21).


 


 

Cause

Faulty Stepping motor driver DR21.


 


 

Remarks

Technically, R33ZTA2080200 is the code for NS6000 and R35N700003600 is the code for NS7000.

However, to avoid any confusion, KG(S) use R33ZTA2080200 to prepare the item with their system.

R35N700003600 is not a mistake and can use it with no problem.

According to Epson both are for stepping motor driver (PMD07C), but only "R33ZTA2080200" is actually used. So pls use R33ZTA2080200, not R35N700003600.
