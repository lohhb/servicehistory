Subject: Simulate the bug of ASES with NS-7000 & NS-6040

Handler Model: NS-7000 and NS-6040

Controller:

Date: 7 Jan 2009


 

Symptom

Current Situation (Bug?)

1.NS7XXX handler allow user to enable or disable the site during normal production upon pressing [RESET] button on operation  panel

(Start mode on MMI is change from Initial Start? to ?Continuous Start?) without prompting any error and forcing user to clear out the units in the handler.

However, handler encountered following error message when it starts.

Error Message : SRC Error

Error Task: 5

Error Func: INARM

Error Line: 2519

Error Joint: 0

Error Code: 2031

Error Message: The subscription of Array variable is out of the user defined range.


 


 

 As for NS6040 handler, error message will pop out and request user to change the start mode to ?Initial Start? when ?RESET? button on operation panel is pressed and site is enable or disable.

Thus forcing user to do a ?CLEAN OUT? on handler prior site enabling or disabling.

ASESG would like to request the software of NS7XXX to be reacting the same manner as NS6040.

Current MMI and SPEL version of ASESG NS7XXX is J7.10-SAS and 2.81b1H respectively.


 


 


 

Action

For NS-7000:

Tried hot mode with different combination of ONE CYCLE/ PAUSE + RESET / Initial start / Site disable/ Site enable in dry no tray.

No error.

Even when running, press pause+reset(automatic switch to continuous start), then change to Initial start also no error as no device is use.

Normally, with device it will give Initial start error.

I guess we won't be able to simulate the problem as we do not have device.

According to Ogata, EPSON tried to simulate the problem but error never occurred.(Not sure if EPSON simulated using device or not).
