Service History

Subject: NS-8080 Index arm device drop & unit stuck at the test socket

Handler Model: NS-8080(NS-8080-02, S/N: 181581)

Controller: RC520

Date: 18~19 Aug 2009


 

Symptom

Reported by customer index arm device drop & unit stuck at the test socket.


 


 

Action

-Check the vacuum units reading which is around 85kpa for both index arm.

-Check the socket tightness by manually putting the device into the socket (no abnormalities found)

-Check the skip enable switch in functioning.

-Check the contactor head (nest) in good condition.

-Dummy device carried out with 3000pcs, there is no double device errors found.

-Teach CS Lee the method to do the HMI upgrading and pass an office release soft copy CD HMI19.17MSM to cristina.

-Check the double device AMP setting -ok


 

After investigation, found that the double device issue is caused by human error. According to the history errors message of handler, the operator never take out the drop device and did not press the "skip enable switch". They just keep pressing the "skip start button" on the operation panel.

-Rerun the double device lot, there is no over size device found.


 


 

Cause


 


 

Remarks

Chamber type handler is using socket layout kit of change over kit.

 This kit don't have double-device detection sensor.

ST-Micro NS8080 S/N 181982 has socket sensor.

Socket senor need designer checking for install enable or not.

This is depend on device size ( thickness ) , socket design, load board design and test head clearance etc... And also change over kit re-design and software modification.

I think that prevent for device stuck in socket = prevent for double device in socket.
