Service History

Subject: NS-7000 SRC error - output arm, TAG_TRY variable overflow" while pressing "START"

Handler Model: NS-7000 (S/N: 18724)

Controller: RC520

Date: 18 Jun 2009


 

Symptom

Encountered error msg in MMI, "SRC error - output arm, TAG_TRY variable overflow" while pressing "START".  Homing no problem.  Reset the handler (off and then on back the main power switch) a few times, to no avail.  Finally, after I HOME the handler, before I START the handler, I push the output arm a bit and then START again.  No problem at all, that is, no more error. Subsequently, cannot duplicate the error again.

Customer ask why error history not captured?  What is this error about?


 

Handler info as follow:

s/n:18724

MFG Feb 2004

MMI v 1.1c10

SPEL 2.81b


 


 


 

Action


 


 


 

Cause

Error happens during continuous start.

Detail of situation

- Changed Bin No after PAUSE + RESET

- Changed Unload direction after PAUSE + RESET


 

Then,

- Push "START" (It is continuous mode.)

- There is device on the output shuttle or output hand brings the device.

- Handler can not judge which place is unload position.

- Indicate the error message


 

How to clear the error;

- Remove all of devices in the handler

- Change to "Initial Start" mode

- Press "START"


 

Note: Our engineer recommends that MMI software is upgrade.

      Ver 1.1c10 is quite old  type.
