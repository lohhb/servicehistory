Subject: NS-8040 HMI temperature offset overwrite by another setup file offset

Handler Model: NS-8040

Controller: RC520 (HMI M11.25-SSY)

Date: 19 Sep 2012


 

Symptom

Last Saturday, STATS experienced temperature offset overwrite by another setup file offset. They didn’t know how it happened, it was complaint by line technician that the devices were having high failure rate due to the temperature out and they found out that the temperature offset was incorrect. Nobody has changed the temperature offset.

Software: HMI ver M11.25-SSY

Tester: Offline Mode

Verification carried out as below:

1) After running Setup A, setup file was changed from Setup A to Setup B . Without running Setup B, the setup file was changed back to Setup A. This was when the Setup A temp offset have been overwrite by Setup B

temp offset. ( Setup A (Running) ----> Setup B ----> Setup A ) - NG

2) After running Setup C, setup file was changed from Setup C to Setup A to Setup B and back to Setup C. Thereafter, Setup C temp offset will be overwrite by Setup B temp offset.

( Setup C (Running) ----> Setup A ----> Setup B ----> Setup C ) - NG

3) After running Setup B, setup file was changed from Setup B to Setup C and did a running. Thereafter, changed to Setup A and back to Setup C. Setup C temp offset was overwrite by Setup A temp offset.

( Setup B (Running) ----> Setup C (Running) ----> Setup A ----> Setup C ) – NG

4) After running Setup B, setup file was changed from Setup B to Setup C and did a running. Thereafter, changed to Setup A to Setup B and the temp offset did not overwrite by Setup A temp offset.

( Setup B (Running) ----> Setup C (Running) ----> Setup A ----> Setup B ) – OK

5) After running Setup A, setup file was changed from Setup A to Setup B and from Setup B to D and did a running. Thereafter, changed Setup D to Setup C and back to Setup D. Setup D temp offset was overwrite

by Setup C temp offset. ( Setup A (Running) ----> Setup B ----> Setup D (Running) ----> Setup C ----> Setup D ) - NG

Below were what I had carried out:

1) Changed temp offset in one of the setup files and checked on other setup files to ensure no overwrite issue. – OK

2) Changed temp offset in one of the setup files, did an ON/OFF of power at control panel and to ensure no overwrite issue. – OK

3) Reinstalled of HMI, deleted all the setup files, recreated the setup files using Epson default setup file and verified. – NG

4) Did a recovery of HDD, created the setup files and verified. – NG

5) Tried out HMI ver M12.52-SSY – NG

6) Tried out HMI ver 2.0 – OK

Note: These issue was very critical to STATS because they are running Intel products and each product has it own temperature offset.


 


 

Action

EPSON released Ver12.56-SSY already solved the problem.
