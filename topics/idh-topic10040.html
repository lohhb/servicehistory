Service History

Subject: Silabs - NS-7000 T4301 Pulse Generating Board detected limit signal issue and I2066 Variable Overflow

Handler Model: NS-7000 (S/N: 18732)

Controller: RC520

Date: 13~14 Apr 2009


 

Symptom

1) Reported by customer over the weekend handler with high occurrence of "Pulse Generating Board detected limit signal".


 

2) Handler release to production, customer report unable to home/start handler. Found out in SPEL showing  I2066 "Variable Overflow".


 


 

Action

1)Check input/output arm's open/close and Z stepper motors. No problem with stepper motors' wirings.

Home handler, found out output arm unable to complete homing due to hand A hand up sensor no feedback.

Re-sit input and output arm stepper motor cable connector, check and tidy wiring on input and output arm. Found output arm hand A sensor signal wire broken. Re-solder wire.


 


 

2)Suspect setupfile corrupted. Deleted and create new setupfile OK.

2nd problem still exist, found out in MMI's contact counter kinds showing 500% instead of showing correct %. Clear contact counter kinds counter. Tried out different setupfiles ¨C OK Problem could be due to MMI having some bugs.


 

Reported by customer 1st problem still exist.

Check PSUNT power box cooling fans working.

Check power box 24vdc for PG1, PG2, PG3 (C-864v1) OK.

Found no abnormalities on 24vdc supply.

Check RC520 cooling fans also functioning.

No error occurred when running check with dummy and dry cycle offline.

But Tester program got problem, unable to run online.

Monitoring problem in progress.


 


 

Cause


 

Remarks

This is related to 0133-NS7K, 0150-NS7K and 0161-NS7K

PG Board C-863 and C-864v1 are not compatible.

PG#1 control Z motion of Input Hand A, B, C and D.

PG#2 control Z motion of Output Hand A, B, C and D.

PG#3 control Open/Close motion of Input Hand and Output.
