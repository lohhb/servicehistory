Service History

Subject: NS-6040 ver3.0c - Boot up failure

Handler Model: NS-6040( S/N: 15484) with CPU upgraded

Controller: SRC520

Date: 7 May 2009


 

Symptom

NST # 16 - Controller during boot up and hang with prompt error message .

 ***STOP : 0x00000078 ( 0xF741B84C , 0xC0000034 , 0x00000000 , 0x00000000 )

 INACCESSIBLE _ BOOT _ DEVICE


 

NST#16 ( S/N: 15484 ) last year upgrading CPU controller with new hard drive , MMI upgrade to 3.0c with windows 2000 . But yesterday facing handler down suspect due to system software corrupted and prompt out error as mention below, after power ON handler for few times , error still remain failed to boot to MMI environment .


 

Re-installed recovery CD and now handler able to boot until MMI environment but prompt out message as attached file below " PCI CARD DRIVERS " , after press " cancel " , handler completely go to MMI environment . Try handler homing but fail for Output Arm torque error .


 


 

Cause


 


 

Remarks

for "INACCESSIBLE _ BOOT _ DEVICE ", I guess could be the following:

1) software corruption

2) HDD going to fail or such as bad sector

3) another HDD from a normal NS-6040( that did not go through CPU upgrade) was plug into this handler

"INACCESSIBLE _ BOOT _ DEVICE " message very broad error.

 

prompt for " PCI CARD DRIVERS , reinstall Contec PCI driver from the MMI CD...maybe in the "Driver"->"Contec" folder.


 

Look for "Multimedia Controller" in Windows 2000 Start->Control Panel->System-> Device Manager, select update driver and select the driver from Contec folder in MMI CD.

 

For "Try handler homing but fail for Output Arm torque error " this may be due to incorrect calibration data of old backup for Output Arm(Robot 3).

Try MCORG for Robot 3.
