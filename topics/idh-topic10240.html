Subject: Silab NS-7000 T-4190 Communication error with module: VSRCPG

Handler Model: NS-7000 (S/N: 18732)

Controller: RC520

Date: 15 May 2009


 

Symptom

After installed PG Boards C-863 into  Silabs NS-7000 (S/N:18732), encountered  spel error "T-4190 Communication error with module: VSRCPG" after the replacement yesterday. So we changed back to original.

Silabs NS-7000 (S/N:18732) PG board revision using is C-864V1.


 

Action

PG Board model C-863 and C-864V1  are NOT direct compatible?


 

When you replace from C-863 to C-864v1, it is replaced not only PG board


 

but also wiring line (PG board to driver).


 

Because, some of the wiring different.

12 June 2009

There are 2 kind of error relation with PG board.

1.      T-4190 Communication error with the module: It happened open / close on the

          Input hand -> PG3


 

2.      T04310 The pulse Generating Board detected a limit signal:

         It was happened hand “D” on the Input and Output hand. -> PG1 an d 2


 

We suspect that these cases are possibility of the reason.

1.      T-4190: (1) PG3 board was broken.

          (2) Connection problem ISA bus on the PG3

          (3) Some of the other hardware may connect I/O port address (600 to 67F).

          PG3 is using I/O port address 600 to 67F


 

       Do you have any additional board into the controller?

  


 

2.      T-4310:

          (1) Connection problem or almost breaking down connector on the

          PG1 and PG2 cable. (PG* 24V-2  No3 pin)


 

          (2) Detect the error fourth axis limit signal on the PG board.

          All of limit signal connect to RGND (0V) by PG* 24V-2 connector. 

          When the point is broken, the error is appeared.

26 Jun 2009  PG board limit signal and communication error investigation by Suzuki san

1. Short 24v but 24v fuse did not trip. Suspect components over-current for limit signal error.

2. Check PSU1 and PSU2 DC voltage at 23.6v. Re-adjust to 24v.

3. Check 24v components, Remote IO, fan driver and 24v fuse current. All within acceptable range.

4. Recommend to replace 24v fuse.


 

29 Jun 2009  PG board communication error

Replacement of 24V fuse, check resistance of all input and output motor wirings, checking of all stepping motor driver settings by Suzuki san.


 

10 Jul 2009  PG board communication error

Upgraded MMI version from v1.1c4 to v1.1c8


 


 

Cause

It seems that upgrade MMI from v1.1c4 to v1.1c8 solved the PG board communication error.


 

Remarks

This is related to 0133-NS7K, 0150-NS7K and 0151-NS7K


 

PG Board C-863 and C-864v1 are not compatible.

PG#1 control Z motion of Input Hand A, B, C and D.

PG#2 control Z motion of Output Hand A, B, C and D.

PG#3 control Open/Close motion of Input Hand and Output.
