Service History

Subject: NS-7000 Chattering error at Index Arm

Handler Model: NS-7000

Controller: RC520

Date: 5 Dec 2012


 

Symptom

SCS feedback that they experienced devices drop at the socket area on their NS7000. After some investigation by themselves, they found out that it was due to chattering error at Index Arm.

Below was their finding.

Model: NS70000

MMI: ver H12.22-SSY

Simulate chattering error at index arm by turning vacuum sensor OFF/ON during the run.

1) When chattering occurred, "Push the Skip Enable switch" message shown instead of "Chattering Error" message.

2) "Skip Enable" + "Skip" buttons came on.

3) Output shuttle was inside chamber.

4) Depress "Skip Enable" + "Skip" + "Start" buttons.

5) Index arm plunge out the devices into the socket area.

6) Output shuttle outside chamber.

7) "Shuttle Device condition error" message occurred.

8) "Retry" + "Skip" buttons came on.

9) Depress "Skip" + "Start" buttons.

10) "Device pickup error at Index arm" message occurred.

11) "Retry" + "Skip" buttons came on.

12) Depress "Skip" + "Start" buttons.

13) "Robot controller (SRC520) error occurs" message occurred.

Why the "chattering error" message was not shown when the index arm vacuum sensor was turned OFF/ON during the run?

Is the sequences correct?
