Service History

Subject: NS-7080W hang up

Handler Model: NS-7080W(NS7KW-06 & NS7KW-08)

Controller: RC520

Date: 6 Sep 2010


 

Symptom

For NS7KW-06

Tester waiting SRQ signal from handler during mid testing, we need reboot handler PC to resume communication.

 


 

For NS7KW-08

Handler hangs up during testing, error show below;

MMI status is PAUSE, pop up error message is "Quick Pause DAEMON 01 Failed. (Error: 7101)"

When error was occurred no respond even we pressed the ‘stop’ button. We need to reboot the handler PC in order to continue production.


 


 

Action

8 Sep 2010 email from Hidaka

 I would like to tell you about patch file of NS-7080W in Xilinx.


 

 1. Current software does not support the contents of patch file.

 So, we will make new installer and support the contents.

 When we have released the software, I will inform you.

 It will be Ver M10.12-SXC


 

 2. Why 7101 error appeared in MMI?

  - Not enough the tolerable amount of event.

 What is changed the patch file?

 ⇒ To change the quantity of supported event  Max event: From 128 to

 256


 

 Therefore, we will not have the problem after used patch file.


 

 Note: How to check the error condition?

    To push "pause+start" repeat twenty to thirty times

    while the handler is running.


 

7 Sep 2010

Xilinx.reg patch applied to NS7KW-08 & NS7KW-06 hang up for Quick Pause DAEMON error. If no more occurrence, will include in MMI version M10.12-SXC


 


 

Cause

Registry problem.


 


 

Remarks

1) Why untill now, EPSON decided to inlcude in M10.12-SXC? What happen in 2008, nobody keep track?

Ans: This is registry setting. So, we did not improve it.

  (Usually, it does not appear in the handler.)

Ans:

This is not support all of standard software.


 

3) What is the unit for this? 128 events? 256 events? or task? or bytes/MB/GB/TB?

Ans:

This is event count. So, this number is quantity.

why not the maximum allowable?

This is limit of events quantity.

  Ex: Current System information

    Create System Count= 192

    Used Event Count = 118

Max Event Count = 255
