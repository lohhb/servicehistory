Service History

Subject: Encountered Error T-4216 "Servo real-time command over-run” during Panel Start or Home.

Handler Model: NS-6040 (UTAC, CPU Board Type B upgrade for NS60-27)

Controller: SRC520

Date: 4 Aug 2008


 

Symptom

After CPU upgrade for NS-6040, encountered “T 4216 Servo real-time command over-run” during Operation Panel Start or Home.

Found out in SPEL, all motors status show ???. Re-start SPEL, Status for all motors seem normal with motor off, but when go into windows explorer, ??? for motor status came back.


 


 

Action

4 Aug 2008

After CPU upgrade for NS-6040, encountered “T 4216 Servo real-time command over-run” during Panel Start or Home.

Found out in SPEL at bottom right of screen, all motors status show ???. Re-start SPEL, Status for all motors seem normal with motor off, but when go into windows explorer, ??? for motor status came back.


 


 

5 Aug 2008

- Replaced 1 new set of CPU upgrade kit, error persist.

- Found that old 2.5 inch HDD already using win98 OS. Tried use back old 2.5 inch HDD with new upgrade kit, no Servo real-time error occur but during homing encounter T-4009 : Timeout error within motion control module.


 


 

6 Aug 2008

Perform manual installation of OS, SPEL 95, MMI 2.0c12 and drivers.

No more T4216:Servo real-time cmd overrun error but encountered T-4009:Time out error within motion control module.

Uninstall SPEL 95 and reinstall but found that installation option can only choose max 600MHZ for CPU clock speed. Normally upgrade by recovery cd, SPEL95 recognize CPU clock speed as 850MHZ.

Tried using 600MHz as clock speed and able to run handler without any problem with the old 2.5 inch HDD after reinstall SPEL95.

Explain to Andrew about situation and he agreed to run the handler over the night in dry cycle mode.


 

No error encountered after running over the night.

Check handler input and output pick and place pose using UTAC reference kit – OK.
