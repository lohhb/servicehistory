Service History

Subject: AMD  NS301 category display installation but LED unable to display

Handler Model: NS-3000 ATC (NS301, S/N: 151024) MMI A9.11-AMD, SPEL CT 2.81b

Controller: RC520

Date: 20 Apr 2010


 

Symptom

1.Replaced top panel, cover tray buffer cover.

2.Installed fix tray display box.

3.Installed RC520 output board(PO-64).

4.Installed Omron LED driver board, cables for PO64-LED board.

5.Installed LED display x6pcs(tray buffer, LD1, LD2, Fix 1,2,3).


 

- Encountered CD-rom drive unable to read contec driver installation disc. Changed CD-rom drive from another UBS handler ¨C OK.

- Encountered windows hang up, mouse cannot move during ¡°New found  hardware¡± wizard after boot up. Replaced another PO-64PCI card from another UBS handler, not OK. Changed PO-64(PCI) card to another available PCI slot ¨C OK.

- After completed contec driver installation, found LED unable to display according to bin. Confirmed SPEL hardware settings output port correct.

Checked Omron LED driver board dip switch S1 and S2 settings at '2'.

Checked NS303 setting at '0'.

Changed NS301 setting to '0' but category still unable to display correctly.


 

Action

24 Jun 2010

-Checking Windows 2000 Input & Output address of [Multifunction adapters] for CONTEC Co., Ltd-PO-64L(PCI)  on LED display, Resources address is "EC00H"

-Set SPEL Output Address of 7segment LED to "EC00H" which is same as Windows 2000 Resources address, restart NS301 and LED display able to function according to Bin setting.


 


 

Cause

Wrong setting of SPEL Output Address of 7Segment(Contec PO-64PCI).
