Service History

Subject: NS-7000 Vacuum Check Height

Handler Model: NS-7000

Controller: RC520

Date: 27 Jun 2008


 

Symptom

Double devices issue on NS-7000 due to the "vacuum check height" value.


 


 

Action

Check on the double devices issue on NS-7000 due to the "vacuum check height" value.

The height position is 15mm for NS7000/NS8000 series. This cause the index arm unable to detect device drop at the socket.


 


 

Cause


 

Remarks

1 Jul 2008

Below information were feedback by SCS Mr Tan Wee Boon.

For item#3, if the software of the NS7000 series handler can detect device stuck issue I am ok but the current software is not able to detect the problem unless the vacuum check height is set to 0 mm (current vacuum check height setting should be 15 mm which we cannot make any changes)

If there is a device stuck at socket during initial start or device drop, this is the outcome.

During initial start – not able to detect device stuck at socket, will continue running.

Device drop before test arm plunge to socket – not able to detect device stuck at socket, will continue running after "skip" button depressed.

Device drop after test arm plunge to socket – able to detect device stuck at socket.

Model : NS7000/7080W

Software version : MMI ver H7.11-SSY
