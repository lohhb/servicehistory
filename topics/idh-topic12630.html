Service History

Subject: NS-8040 T4190-Communication error with the module:VSRCPG

Handler Model: NS-8040(NS80-26, S/N: 181540)

Controller: RC520 (S/N: 03-01164)

Date: 29~30 May 2010


 

Symptom

Informed by UTAC Ray handler encountered this error during production.

Initially HMI unable to launch.

Exit HMI launching and Initiate SPEL CT but encountered error T-4190 Communication error with the module:VSRCPG

.

Advise Ray to perform recovery with recovery 1.56 and HMI 1.5, restored system, pose and setup files but error still persist. TBC on Monday.


 


 

Action

31 May 2010

When finished recovery using 1.56 recovery disc, HMI unable to launch.

Close HMI and activate SPELC T thru desktop icon and error will appear.

Swapped another 2-in-1 PG board NG.

Swapped another CPU board OK.

Replaced back original CPU board OK also.

Dry cycle run and monitor OK.


 

1 Jun 2010 I just received reply from Japan.

1.       Remove the all of PG board from controller and remove the cable.

          Install PG board and cable.


 

2.       If 1. process is done but it still has the problem,

          Replace all the PG boards.


 

   Note: It is cleared after CPU board reset.

             This was heard from you.


 


 

Cause

Electrical contact for CPU Board and Backplane board not good, problem solved after resit CPU Board.
