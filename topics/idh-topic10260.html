Service History

Subject: NS-7000(S/N: 18825) Input Shuttle 1 & 2 position out

Handler Model: NS-7000

Controller: RC520

Date: 14 May 2009


 

Symptom

Install SPEL CT 2.81b, PGEdit 2.2a, Install MMI 1.1c.

Restore system data.

Home handler, ok.

Shuttle 1 & 2 position out, reteach but checking by SPEL  for jump P22+X(60), shuttle 2 only move by 40mm. Same problem for shuttle 1.


 


 

Action

The problem is when robot 6 jump p22+x(60), input shuttle 2 only move by 40mm during SPEL command checking, same problem also for shuttle 1. The problem is because when put changekit into the shuttles and press START button, handler will check shuttle 1 & 2, and Device presence sensor detect that there is device in Input shuttle 1 & 2 pockets. Condition of shuttles sensors already checked, ok.


 

So the next step I checked was using SPEL CT, so execute commands to check P22 position and P24, ok. But when executed jump p22+x(60), input shuttle 2 only move by 40mm. Same problem for shuttle 1.


 

Handler dry run without changekit ok but put changekit then got problem.


 

Problem not solved yet for S/N: 18825.


 

Used a clone backup from NS-7000(S/N: 18724) (Wide Hotplate) and restore to NS-7000 S/N: 18825, and configure to normal hotplate.

Restore SPEL system and pos.


 

Home and start to run handler with shuttle kit and  same problem as last week : initial check by MMI shows device in input shuttle 1&2.


 

Just changed the position of Device presence sensors for Shuttle 1 & 2 and problem solved.


 

Now dry run handler.


 


 

Cause

Suspect the MCD setting like gear ratio or pulse to mm setting.
