Subject: NS8080-01(181665)  War02223 ¨C RS485 point file data error + output arm hand up sensor doesn't turn on error

Handler Model: NS-8080 (NS8080-01, S/N: 181665) 8 Hands High UPH

Controller: RC520

Date: 20 Dec 2010


 

Symptom

UTAC  NS8080-01(181665)  War02223 ¨C RS485 point file data error + output arm hand up sensor doesn't turn on error.


 


 

Action

1.When handler power up and boot into HMI, War02223 ¨C RS485 point file data error. Unable to clear the error, press home or start also no response.

2.Tried to perform hand height calibration for output arm but encountered RS485 Init error.

3.Check SPEL and found RS485_ALN task hanged.

4.Quit HMI and open z-teach but encountered file open error.

5.Check pose data in z-teach and found no data for hand c,d,e,f,g,h.

6.Next, clicked the "Download" button but nothing happened.

7.So after reboot again, handler able to home and checked that the output arm has pose data.

8.After started dry cycling the handler, encountered output arm hand up sensor doesn¡¯t turn on. Previously already encountered 2 times for this issue, re-sit connectors ok but it came back again. In SPEL input bit IO #52, the light will turn off at the output shuttle 1 and 2 position. Sometimes when we wait for a while without touching the hands, the light will come back ON. When I move the top covers on the 8-hand assembly, the light will flicker On/Off.

9.Loosen up all the screws on the 8-hand assembly, run the handler in low speed and slowly tightened the screws one by one at the output shuttle position, making sure the IO#52 is light ON always. After all the screws tightened and covers put back, handler continue to run without error.


 

23 Dec 2010

1.Replaced new spare control board(sent from Japan for EEMS in June).

2.Check fuse on old Control Board and new Control Board, Old Control Board Fuse open.

3.Dry cycling handler and check if any flickering on IO #52. Ok.

4.Release handler and to confirm again handler status the next day.

5.Raise warranty report S0462.
