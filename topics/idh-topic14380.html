Service History

Subject: NS-7000 ESD class 0 upgrade Windows unable to detect RS485 card.

Handler Model: NS-7000 (NS70-34, S/N: 181008)

Controller: RC520

Date: 8~9 Feb 2011


 

Symptom

UTAC  NS70-34(181008) ESD class 0 #6 upgrade.

Completed all hardware installation.

Encountered windows unable to detect RS485 card in RC20 controller PCI slot.


 


 

Action

Swapped RS485 to IFUNT (GPIB) PCI slot, windows able to detect new hardware But IFUNT card swapped to RS485PCI slot, IFUNT100 interface unit(behind handler) all LED light up which was abnormal.

Sometimes MMI cannot boot up and pc performance will lag.

Suspec PCI slot on backplane board faulty.

Awaiting advise to use another RC520 or backplane board.


 

9 Feb 2011

Swapped backplane board from loan NS70-99c to ESD class 0 upgrade handler NS70-34, but the windows still unable to detect the new RS485.

Swapped another handler NS70-42 HDD over and able to detect the card.

Performed recovery on the old hdd, restore backups and RS485 detection problem resolved.


 


 

Cause

Old HDD maybe corrupted or due to some software issue unable to install software driver for RS485 card.


 


 

Remarks

Backplane is Contec MB-PCI14/9(PC) for NS-7000 which is also use in NS-6000.

Backplane is Contec BPI-1411 for NS-8000.
