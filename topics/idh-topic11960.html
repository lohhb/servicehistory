Subject: NS-8040 after RC520 bootup HMI Windows only display NS taskbar and executing SPEL, hang up

Handler Model: NS-8040 (NS-001)

Controller: RC520

Date: 11 Mar 2010


 

Symptom

RC520 bootup, only display NS taskbar and shows executing SPEL.


 


 

Action

- windows task manager stop SPEL CT and HMI

- launch SPEL ok

- launch HMI, many pop up windows shows unable to read file in RUN folder of NS8000

- re-install HMI & reboot also cannot for several times.

- launch SPEL with many error reading files in RUN folder

- delete RUN folder in C:\NS8000, and copy & paste backup back to c:\NS8000  but still encounter errors reading files in RUN folder

- change to different setup file, no more error

- suspect setup file (V93K)PG-BGA323-D-AMB corrupted

- delete  setup file (V93K)PG-BGA323-D-AMB

- import  setup file (V93K)PG-BGA323-D-AMB from backup

- dummy devices run with  setup file (V93K)PG-BGA323-D-AMB, ok


 


 

Cause

Corrupted setup file caused RC520 bootup HMI Windows only display NS taskbar and executing SPEL, hang up.
