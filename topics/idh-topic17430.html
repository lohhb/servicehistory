Service History

Subject: NS-7080W Drive Unit 2 Input & Output Arm encountered F5001: Encoder signal cable disconnection

Handler Model: NS-7080W (S/N: 121223, NS7080-05) UTL

Controller: RC520

Date: 31 Jan 2012 to 3 Feb 2012


 

Symptom

UTL – NS-7080W(121223) DU 2 input and output arm encountered F-5001: Encoder signal cable disconnection.

Reported input arm done calibration on the 17/Jan and after shutdown and power up on the 20/Jan, encountered encoder signal disconnection error and SPEL robot 2 status showing '???'. Followed by output arm(robot 3) encountered same issue the next day.

DU slot 2 displayed status 1 error led.


 

Action

1. Checked DU lithium battery voltage 3.6v ok.
2. Checked all other robots able to motor on.
3. Swapped DU2 from NS7080-04 to NS7080-05, same issue.
4. Swapped NS7080-04 RC520 controller to NS7080-05, same issue.
5. Swapped DU2 motors power and encoder connection with DU1. Both input and output arm X-axis showed '???' on DU1. Problem migrated over.
6. Performed reset encoder and calibration of input/output arm XY axis and home handler ok.
7. After shutdown handler and reboot again, encountered same issue on input arm.

The encoder signal cable was detected to be disconnected either at Phase A or Phase B.

Check on the M/C signal cable connection.

Check on the signal cables inside the Manipulator for any loose pins, cable disconnection or short-circuit.

Replace the motor in case of the encoder failure.

Replace the Drive Unit in case of DMB or Power unit

failure.

Check on the cable connection between the control and peripheral units; Check on the connectors:

EMERGENCY, D-I/O and Expansion I/O; Check on the short-circuits with the connecting cables or/and inside the peripheral units.


 

Initialise & calibrate Input & Output Arms:

- (for DU2/Slot2) set DIP switch Bit 1,2,3,4 all to ON

-press & hold white RESET button for 3 sec or more (this will reset the encoders’ of the motors)

- (for DU2/Slot2) set DIP switch Bit 1,2,3,4 all to OFF

- in SPEL and switch to “P” mode , “Abort all” then close SPEL and Restart SPEL

- SPEL is launched, switch to “P” mode press “Abort all” then should see pop up message “F-5136 Absolute Encoder has been initialized”.

-make sure correct robot is selected, eg in SPEL Command Execution Windows, select selrb 2 then type "reset".

- Robot 2 with X & Y move to mechanical pulse 0 position

- calpls 0,0,0,0 and calib 1,2 (in this case for Axis 1 and 2)

-make sure correct robot is selected, eg in SPEL Command Execution Windows, select selrb 3 then type "reset".

- Robot 3 with X & Y move to mechanical pulse 0 position

- calpls 0,0,0,0 and calib 1,2 (in this case for Axis 1 and 2)


 


 

Cause

Input arm X-axis 400W motor  and 400W servo driver are spoilt.
