Subject: NS-8040 Index Arm device pickup error,  low vacuum reading at Index

Handler Model: NS-8040

Controller: RC520

Date: 3 Jun 2010


 

Symptom

At Infineon doing NS-8040 buyoff using PG-WF2SGA-121-1 Quad 6x6x0.73 change kit.


 

During AutoContact Height, some devices were able to pickup and some devices dropped, vacuum reading for devices that were able to pickup at Index shows -40kpa to -68kpa.


 

The pickup tip for the Index Contactors were all metal plunger, no rubber cup.

I guess because the device was too small for rubber cup.


 

After change the default Index vacuum sensitivity from -50kpa to -35kpa, able to do AutoContact height and running with devices.


 


 

Action

FYI,  we starting encounter vacuum low/leak issue at early year 2008 for PG-WF2SGA-121-1 and PG-WF2SGA-143-1 single site kit.


 

Then we found previous plunger design (without suction cup) which able to run in NS6040 and NS7000 cannot apply to NS8040 / NS8080.


 

We make improvement design to longer plunger to close the gap between plunger and device to prevent vacuum low/leak issue.


 

The result show the vacuum reading increase to -60kpa to -70kpa for the new plunger design compare previous plunger design -40kpa.


 

 Then the latest plunger design (longer) apply to all Infineon change kit (small device without suction cup):
