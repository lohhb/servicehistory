Subject: NS-8040 flying device(device condition error) at Output Shuttles for WFWLB-183 / 217

Handler Model: NS-8040

Controller: RC520

Date: 13 Apr 2009


 

Symptom

Flying device at Output Shuttles WFWLB-183 / 217 package.

Device condition error at output shuttle.

WFWLB217-1_Q_AMB (device size 8x8x0.725).


 

Action

At Infineon helping one of the setup tech to do pick & place offset for the package (LTX)_WF2SGA143-1_S_AMB.This is a single site kit, device size 6x7x?, don't know what is the thickness. There was not device flying at Test Arms.

 

Where as last week, the same NS-8040, running Dual site kit  (LTX)_WFWLB217-1_Q_AMB or Dual site kit have encountered device flying or dropped at Test Arms. Device size 8x8x0.725.

Possible reasons for flying device was that the purge air was being adjusted, the other could be the contact height and device could be sticky when using rubber cup at Test heads.

 

Not sure if the nest/contactor for both kit of of same materials or same design....bcos I forgot to take out the contact for  (LTX)_WF2SGA143-1_S_AMB today.

Basically, I just want to know if you got any idea what caused the device fly away?

Any design change(s) to be made to the  WF2SGA143-1 dual and Quad site kit?


 

WF2SGA-143 the thickness of device 0.73mm

Both material for WF2SGA-143 and WFWLB-183 / 217 is same SEMITRON 520 ESD HR.

 

Basically, the design concept for both nest is the same.

The only difference is WF2SGA-143 without suction cup and WFWLB-183 / 217 come with suction cup.

The suction cup used mostly to encounter the air leak issue.

 

But i don't think is the suction cup used cause the device flying issue.

Because the Infineon C/K TF2BGA-389 using the same suction cup without any device flying issue.

For other test house, we also using same suction cup for even smaller and thinner device like 5x5x0.55 BGA without device flying issue as well.

 

So, we suspect either the purge air non-consistent, or device property sticky.


 


 

Cause


 

Remarks

For change kit with shuttles' pocket depth is very shallow, after Auto Contact Height, need to raise up contact height by 0.5mm to 3mm higher to prevent flying device.

eg. WFWLB-183 / 217


 

For Output Device Condition error, check points:

1) Redo Auto Contact Height (after AutoContact Height increase the device release height by 0.5mm to 3mm). Can try reduce device release height to output shuttle.

2) Index Counter Air On Time (increase or decrease?)

3) Remove Output Shuttle jig, check below the output shuttle plastic if it is dirty or have device

4) For Square site(eg. Quad, Octal..etc), check Gate Sensor(eg Receiver) is working properly and it's position

5) Output Shuttle Sensor Board (SKP456C, SKP456D, SKP456E)

6) Gate Sensor Board (Shuttle board)

7) For 1x2 or inline4, Output Shuttle Device Detection sensor(horizontal) alignment & working condition

8) Index Nest (worn out? such as chip off?)

9) O-ring worn out on plunger (harden? broken?)

10) Index Counter Air (purge air) volume eg. is it too strong or weak

11) Plunger in the contactor(dirty? plunger able to spring out?)

12) Spring in Contactor (is it dirty? in good condition?)

13) Rubber cup in Contactor (sticky? that stick device? harden? worn out?)

14) Wire connection on sensor(eg. broken wire, connector connected properly to sensor board?)

15) Input I/O Module for EIN10 (SHUTTLE3) / EIN13 (96) near to Gate Sensor Board(Shuttle Board)

EIN 10 16CH for NS-8020, NS-8040

EIN 13  8 CH for Timing sensor IN148, IN149 (For NS-8080, NS-8160)
