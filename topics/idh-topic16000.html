Service History

Subject: NS-6000 Unable to install MMI X12.10-SAS from CD.
“NSSETUP.exe has generated error and will be closed by Windows.”

Handler Model: NS-6000 (S/N: 12469)

Controller: SRC520  (CPU Board SPI-6941, MMI 3.0e  SPEL CT 2.81b)

Date: 5 Sep 2012


 

Symptom

AMD – 2D Bar Code Software X12.10-SAS installtion to NS-6000 (S/N: 12469)

Unable to install MMI X12.10-SAS from CD.
“NSSETUP.exe has generated error and will be closed by Windows.”

Handler installed with ATC for site 1 on both Index Arm.
Backup existing data in controller.


 

Action

Copy files from CD to USB thumbdrive.
Install MMI from CD, ok.


 Press START, MMI status changed from RUNNING to PAUSE. Handler pause.
In SPEL, found that in Cont.prg, command execution stop at line “ROPENT q NsRootDir$ + “RUN\2DRoffset.dat” as #DIR””
In MMI OFFSET, change 2D barcode offset for Shuttle position to 0.01, ok.
This is software bug.
Time stamp error encountered for COMDLG32.OCX and TABCTL32.OCX, resolved by changing the date and file size in ver.dat.
After tray with device feed to pickup area, Input hands above shuttle and Error code 50, sub code 7; Decode Unit Initializing error.


 Was informed to change C:\NS6000\CONFIG\Codereader.def, changed COM Port No to 2 from 1.
(For NS-8000, Heater comm using COM2, COM1 for 2DBarCode.

 For NS-6000, Heater use COM1 on SRC520 and COM2 for 2Dbarcode)


 Input Shuttles scanning position for site1, site2 and dual sites for 2D code reader are incorrect.
Decode unit initialisation error.
KG(S) trigger sub-con programmer to come to AMD(S) on 10 Sep for Decoder and protocol issue.


 


 


 

Cause

“NSSETUP.exe has generated error and will be closed by Windows.” could be CD ROM drive is faulty.
