Service History

Subject: NS-6040 CPU Board upgrade problem

Handler Model:

Controller:

Date: Jul 2011


 

Symptom

Amkor(philippines) is continuing doing their CPU upgrade project for NS6040 handler from (200 MHz to 850 MHz), they encounter some incompatibility between the CPU board and SRC-Mother board(backplane board).

Notice that one LED is not  light (3V supply) on backplane board MBP-PCI14R when new cpu board SPI-6941 is used. Even peripheral boards are not yet installed.


 

Effect of no +3V supply, Using NEW CPU board:

a) IFUNT100 cannot be detected as well as other PCI bridge device for Device Manager

b) 3V in mother board is not light on backplane board.


 

The old cpu board before CPU upgrade, the backplane board, notice that all power indicator LED is light and all peripheral boards are installed. Compared to new CPU board (SPI-6941-LV), +3V LED is not light up even without peripheral boards installed


 

They are using new CPU board which is CPU (SPI-6941-LV), and they have some inquiry about it, attached is the report they created on this issue.

• Need to know if new CPU boardSPI-6941 has a compatibility issue on mother board?

• Need to know if new CPU require higher power supply wattage

• Need to re-confirm jumper setting of New CPU (SPI-6941-LV)

• Need to re-confirm BIOS setting of New CPU (SPI-6941-LV) during installation

They also want to know if this is the same CPU board that is compatible to all SRC.

 If not, they need the quotation for the new CPU board that will compatible to all.

1. Backplane board use for upgraded CPU(SPI-6941-LV) is CONTEC MB-PCI 4/9(PC) while Mother board use for old CPU (P5/6x86 SBC ver. G5) is MBP-PCI14R.
 2. Power supply requirement for CPU(SPI-6941-LV) is 200 watts minimum.



 

Action

Suspect existing 200W PC power supply in SRC520 is not in good condition.

Should replace to a 300W power supply unit.


 


 

Cause

specification: MBP-PCI14R (R35N600000100)


 

another old type Backplane is SRC520CE is MBP-PCI13L
