Service History

Subject: NS-7000 input hand C stepper driver burnt, Input Arm Hand C home sensor error

Handler Model: NS-7000(NS70-13, S/N: 181067)

Controller: RC520

Date: 15 & 16 Apr 2010


 

Symptom

Input hand C stepper driver burnt.

Input Arm Hand C home sensor error.


 


 

Action

15 Apr 2010

1.Measured all stepper driver wiring no internal short or short to ground.

2.Check input hand C stepper driver fuse ok. It can mcal but jerky when hand height calibration and will lose torque.

3.Found input hand A when mcal will not move but able to complete hand height calibration. Did not suspect stepper motor faulty. Use output hand D driver also same symptom.


 

1) Checked stepping motor drivers 1 by 1.

2) Shortage of x02 stepping motor drivers, borrow from office NS-7000.

3) Performed Mcal in SPEL and hand height calibration on both input and output arm ¨C OK.

4) Auto run less than 5 mins, input arm hand c home sensor off error happened again, resulting hand c loosing torque and burnt motor driver.


 


 

16 Apr 2010

1) Checked wiring condition from actuator hand to input arm manifold and to distribution board SKP430.

2) Checked PG cables from PG board to distribution board SKP 430 and found out PG1 cable connector burning smell.

3) Borrow PG board C-873, PG1 cable and stepping motor driver from Office NS-7000.

4) Replaced onto handler, performed MCAL and hand height calibration ¨C OK.

5) Cycle run handler in 20% and 100% speed for 45mins ¨C OK.


 


 

Cause

PG Board 1 cable connector J1 burnt.
