Service History

Subject: T-4241 The detected speed at the Low-power state is too fast occurred intermittently on Shuttle 2

Handler Model: NS-7000 (S/N: 18606)

Controller: RC520

Date: 10 Aug 2010


 

Symptom

Intel (M'sia) NS-7000 encountered T-4241 "The detected speed at the Low-power state is too fast" intermittently on Shuttle 2 for the past 6 months.

Already checked motor, belt tension, servo driver, drive unit and slider motion smooth.

Advised Sean to patch MCD file, update manipulator configuration, tighten belt tension despite same as shuttle 1, swap linear slide for error migration to shuttle 1.

To wait for system time to try and monitor accordingly.


 


 

Action

MCD 06 file replaced and manipulator settings updated.

Found both shuttle 1 and 2 N.G upon position jump in SPEL.

Original MCD file returned and manipulator settings updated.

Still found shuttle 1 and 2 N.G upon position jump in SPEL.

Both scenario, found both having device exist on loading stage and pocket is not in line with device present sensor.

Restored original MCD file again but this time update manipulator with direct drive for shuttle motor.

Only T-4241 "The detected speed at the Low-power state is too fast"  remains. Other errors OK.

To send back original MCD for 05 and 06 to edit for LP monitoring to be OFF and try.


 

17 Aug 2010

NS7000 has shuttle design changed after 2003. Default selection for Manipulator configuration for shuttle motor is with reducer but depending on mechanical hardware to select accordingly.

To edit existing MCD file for 4241 error.

For INTEL M'sia, shuttle without reducer so cannot select with reducer when update manipulator.


 


 

Cause

For INTEL M'sia, shuttle without reducer so cannot select with reducer when update manipulator.
