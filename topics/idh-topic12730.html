Service History

Subject: EEMS NS88-04 S/N:181655 output arm unable to HOME. 2Ack error in Z-teach and "No information" HMI error.

Handler Model: NS-8080 (NS88-04, S/N: 181655) 8 Hands High UPH

Controller: RC520

Date: 18 Jun 2010 ~ 7 Jul 2010


 

Symptom

EEMS  NS88-04 S/N:181655 output arm unable to HOME. 2Ack error in Z-teach and "No information" HMI error.


 


 

Action

1.Replaced Output Hand E.

2.Replaced distribution board SKP479_2A.

3.Replaced control board.

4.Replaced distribution board SKP479_2B.

5.Replaced RS485 board(Serial I/O Board, R35N800022500).

6.Replaced all connection cables.(cables are for input side, length is different. Need to change back to original)

After all the above replacement, problem still exist. Awaiting Hidaka san to send driver module and requesting whole output arm 8-hand assembly.


 

it seems that we need driver board for Hand E:

 

1) Home sensor actually goes through driver board. Pg175, SKH0870-F36

2) Simulate 1 hand no "HOME" on input arm, whole hand actually also no "HOME" and has same error message of "No Information".

3) Driver may be the cause of Hand E vibrating.

 

So we feel strongly that we need the driver board for Hand E. In an event driver board N.G,

it may cause unable to "HOME" and vibrate like what you see now.


 

7 Jul 2010

Changed back to old set of connection cables because the length of new set is different.

Replaced output arm Hand E driver board unit with encoder board.

Handler able to HOME in HMI and Z-Teach. SPEL CT input bit #52 (for output Hands HOME) also can ON after HOME completed.

Ran dry cycle mode 100% OK. Release handler to customer and monitor. Warranty S0435 raised for driver board x2 and encoder board x2.


 


 

Cause

Faulty Driver Board SKP480 and Resolver(Encoder) Board SKP481.


 


 

Remarks

Driver Module (Driver Board SKP480 with Resolver(Encoder) Board SKP481)

Part Code: R33N800014700 + R33N8000148000
