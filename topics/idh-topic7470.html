Service History

Subject:  “group variables not defined” and “device floating error”, handler hang up

Handler Model: NS-6040(CARSEM)

Controller: SRC520

Date: 4~7 Aug 2008


 

Symptom

4 Aug 2008

- Initial error, “group variables not defined” and “device floating error”, handler hang up.

- Recovery CD, MMI and SPEL version up.

- Restore SPEL project and system files.

- “File definition access error”, “RS232C initialize error” and “input arm motion control module time up error”

- Suspect data dated 2004 and 2006 not compatible with current SPEL version leading to errors.

- Need to know original SPEL version back up data is in.


 

5 Aug 2008

- HDD Recovery again but with NO RESTORE of project and system files.

- Just load Drive Unit data.

- Encounter “The distance from the home sensor to the encoder Z phase varies significantly from the previous value” for input arm.

- This error is normal as no restore has been done. Motor data is wrong.

- MCORG all robots and axes.

- Now no more error. Just point teaching is out.


 

7 Aug 2008

- During teaching, found S1 brake switch unable to function.

- Customer measured supply, 24V.

- S1 and S2 connectors swapped, X34 and X44 respectively

- Problem migrate to S2.

- S1 and S2 connected to axis z motor, unable to work.

- Suspect brake system within motor N.G
