Service History

Subject: VBLD project for UBS#17 at Output Shuttles area

Handler Model: UBS #17

Controller: RC520

Date: 22~26 Aug 2011


 

Symptom

AOI (previously VBLD) project for UBS#17 at Output Shuttles area.


 


 

Action

COM-2(PCI)H (RS232 2-port card), cable installed to RC520.

RS232 conflict problem found causing MMI hand up after reboot.
 - Change COM-2(PCI)H card PCI slot, problem solved.

 Do not use PCI-4, PIO64 card on PCI-1

IFUNT200 card on PCI-2,

COM-2(PCI)H card on PCI-3
 -protocol checksum calculation is clarified with AMD. Vision PC communication software has been modified to align with Epson.
-UBS software Ver 11.15-SAS installed but has errors and can not run properly


 -Remarks: Vision camera x02 hardware provided and installed by AMD for Ball Damaged Check at area between Output Shuttle 1&2
 Camera(with blue led ring light) above Input Shuttle with 2 functions(2D Reader & Die crack check)
 Camera site1 at nearest to Output Shuttle. Camera site0 nearest to Fix1 tray area.


 

It is found the new UBS software don’t have X,Y,Z pick & place OFFSET for user to set and adjust in MMI Offset GUI.
In SPEL CT, manually teach Robot 3(Output Arm) Point 10, 11, 12 to set the teaching point (device central position in ball damage detection point as Point 10. It includes center point x,y value. Z set to 0).
Point 10: Both Output Hands above both camera,Z=0
 Point 11: Output Hand 2 above site1 camera(Output Arm nearer to Index), Z=0
 Point 12: Output hand 2 above site0 camera(Output Arm nearer to Fix1 Tray, Z=0


 


 

Remarks
-AMD tester communicates through RC520 CH1
 -AMD 2D PC communicates through RC520 CH2
 -Robot 2(Input Arm) X positive Pulse limit changed from 18800 to 20000
 -In SPEL CT Output IO Board setting, 7SegLED change to address E000 due to unable to display Bin number in Display.

7SegLED must be same as Windows 2000 Device Manager Contec PO64 PCI card.
