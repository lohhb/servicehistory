Service History

Subject: NS-7000 ESD class 0 failed input, output and testsite

Handler Model: NS-7000 (NS70-30)

Controller: RC520

Date: 14 Feb 2011


 

Symptom

UTAC – NS70-30 ESD class 0 failed input, output and testsite


 


 

Action

- check positive/negative decay time and ion balance with ION CPM in chamber, ok. All location passed within specs. 1000v to 50v under 10secs.

- manual trigger MKS controller and check positive/negative decay time for S1, S2 and S3 sensor, ok. All passed decay within 7secs and ion balance within +/-5 volts.

- found chamber ionizer pins not so good condition. Advise to change and improve the frequency of cleaning the pins of chamber and blower fans.

- encountered file reading error when clicked on MMI ESD parameter, checked config folder and “ESD class 0=3” not found in security_new.def file. Removed “fan out.bat” file from startup menu so that when handler reboot, the fan out.bat execution file won’t overwrite the “ESD class 0=3'” settings. When this setting is not set properly, MMI ESD parameter decay time check and ion balance check may not work and handler will not prompt alarm if either of them fails. Advise to check other esd class 0 handlers also.

- found chamber MKS S1 sensor loose. the 2 countersunk screws unable to tighten the sensor to the bracket properly.

- found chamber kasuga controller always ON when chamber fan not running. power cycle handler ok.


 


 

Cause
