Service History

Subject: "Index vacuum unit P2 setting" Chattering error of NS-8040

Handler Model: NS-8040 (S/N:181207)

Controller: RC520

Date:


 

Symptom

During buyoff, usg Quad site kit with adaptor for M2900 package, after Clean Out or One Cycle, Index Arm 1 check on socket cause Device chattering error, Retry, ok after that Index Arm 2 check on socket also Device chattering error.


 


 

Action

- Removed c/k from Index Arms, dry run, no error.

- Install only c/k for Index Arm 1 dry on, chattering error for Arm 1.

- Install only c/k for Index Arm 2 dry run, no error

- In Mode-Up, P2 setting, change value 6 to 10 also have chattering error,

 increase P2 to 15, chattering error occurred one time.


 


 

Cause

Due to old design of Index vacuum, need to increase P2 to 40 to prevent chattering error at Index after Clean Out or One Cycle.


 


 

Remarks

17 Dec 2009 email from Serizawa Hiroaki through Ogata-san email:

Subject:  Technical information for "Index vacuum unit P2 setting" of NS8040

<Contents>

Countermeasure for "INDEX unit device pick-up chattering error (J0309 and J0310).

This trouble happen on condition that NS-8040 manufactured in 2006 to 2007 and when install common software of NS8040/NS8080.

NS8040 handlers manufactured in 2006 to 2007, chattering error occurs when the Index vacuum unit P2 is turned ON.


 

Countermeasure is P2 value change from +6 to +40.


 

Please see attached PDF file and please inform to your service member.

<<e_NS8000_Index Vacuum Sensor_P2.pdf>>


 


 


 

The  reason: Some of the handler wiring P2 output and wrong assignment in uniwire.

                   P2 connect to Uniwire 

                   But uniwire assign some of wrong.


 

Temporary solution: Set P2 more than 40 KPa

                  -> Please change the value especially the occurred handler


 

We tried in our factory handler. We found some of the handler is reached around 36Kpa.


 

P2 not used but is wired. To remove wire as advised by EPSON

New NS8080 and NS8080H have the wire removed from factory side

EPSON has conclusion on the P2 issue for index arm.


 

Solution is some changes to hardware I/O module dipswitch settings Hidaka-san feels it is easier to change current P2 settings to 40.

So, in view of any potential customer questions,

1) All new handlers will now be set to 40 for P2.

2) Any new vacuum unit purchased, japan will set it to 40 for P2.

3) In this case, if new unit changed, it will already be set to 40 for P2.


 

For us:

1) Current handler to change P2 value to 40.
