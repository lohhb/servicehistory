Service History

Subject: Error WAR1701, UniWire 1  0V Short

Handler Model: NS-8040 (UTAC, NS80-03)

Controller: RC520

Date: 23 Jan 2008


 

Symptom

Error WAR1701, UniWire 1  0V Short.


 

Action

1) MMI shows Error WAR1701, UniWire 1  0V Short.

Re-initialize Remote I/O Board.

Check Remote I/O board for physical burnt.

Trace to E OUT I/O module (below Empty Tray buffer).

Check resistance between wires (wires Black, Red, White, Green) of UniWire connector that connect to Remote I/O Board.

Resit all uniwire connectors that are below the loaders and unloaders, ok.


 

2) MMI shows homing...., but Input Arm unable to Home, while the rest of the robots already HOME.

Normally, when Power Off, pull and release the any Hand of  Input/Out Arms' Hand, the Hand would automatically pull up by the spring to it's highest position.

When Power on when Hand is pull down there is a resistance and after that that Hand will remain at that position and will not be pull up by it's spring.

Found Hand A cannot Home.

Power Off, pull Hand A found that there was no resistance from it's stepper motor and when release Hand A, it is automatically pull up by it spring in a very smooth motion.

Swapped connectors for Stepper Driver DRV12 & DRV13, Power On, Hand A still can be pull up by it's spring.

Swapped connectors to stepper motors for HAND A, HAND B. Power On, pull down and release Hand A, the pull down movement was with resistance and Hand A did not spring up. Pull Hand B, found that there was no resistance from it's stepper motor and when release Hand B, it is automatically pull up by it spring in a very smooth motion.

Swapped stepper motor connector between Hand A and Hand C, now the problem swapped over.

Return connectors' connection for Hand A and Hand C.

Input Arm X and Y Axes wiring check, found X1 connector loose on SKP456F board(above Input Arm Remote I/O modules).


 


 

Cause

Loose connection of X1 connector caused WAR1701, UniWire 1  0V Short and Input Hand A no torque.
